## Why This Exists

CoYYns are the fake currency that Yahya and Yara earn, spend, and gift each other throughout Hayah. Every time a user glances at a card, a header, or a dashboard section that involves CoYYns, they need to see their balance instantly — without navigating anywhere. The CoyynsBadge is that instant glance. It's small, honest, and warm. It says: "you have 1,240 CoYYns right now." No decoration. No drama. Just the number, wrapped in a soft pill that feels like it belongs on the page.

This badge appears in headers, inside feature cards, and anywhere a compact balance display is needed. It must be lightweight enough to drop in anywhere without layout consequences.

---

## User Story

As Yahya or Yara, when I look at a page that involves CoYYns, I can see my current balance in a small, legible pill badge without navigating to a dedicated balance screen. The number is formatted clearly, the badge fits naturally in any context, and it updates if the balance changes.

---

## Full UX Flow

1. **Badge renders** → shows coin icon + formatted balance number in a soft pill
2. **If balance is loading** → shows a subtle pulse animation in place of the number
3. **If `balance` prop is provided** → displays that value directly, no hook call
4. **If no `balance` prop** → reads from `useCoyyns()` hook automatically
5. **On hover** → badge scales up slightly (scale 1.02), feels interactive without being clickable
6. **Number always formatted** → commas via `toLocaleString()` (e.g., 1,240 not 1240)

## Implementation Notes

- `"use client"` directive required (uses hook and Framer Motion)
- If `balance` prop is `undefined`, call `useCoyyns()` to get `{ balance, isLoading }`
- If `balance` prop is provided (even if 0), skip the hook entirely — do NOT call `useCoyyns()` when a value is passed
- Format with `balance.toLocaleString()` — no decimals, commas for thousands
- Loading state: show a small animated pulse placeholder instead of the number
- Use `motion.div` with `whileHover={{ scale: 1.02 }}` and `transition={{ duration: 0.15 }}`
- Use `cn()` from `@/lib/utils` for conditional class merging
- Size variants: apply different icon size and text size classes based on the `size` prop
- No click handler — this badge is display-only, not interactive beyond hover scale
- Use `font-mono` Tailwind class for the number (maps to JetBrains Mono)

---

## Edge Cases

- **Balance is 0** → show "0" not empty — zero is a valid balance
- **Balance prop passed as 0** → still display it (do not fall back to hook for falsy check — use `balance !== undefined`)
- **Very large number** → `toLocaleString()` handles this gracefully (e.g., 1,000,000)
- **Hook still loading** → show pulse, not "0" or "undefined"
- **Hook errors** → show "—" as fallback if balance is null/undefined after load
- **sm size in tight layouts** → must not overflow or wrap — test at 60px container width

## Do NOT

- Don't call `useCoyyns()` when a `balance` prop is explicitly provided
- Don't use a falsy check to decide whether to use the prop — use `balance !== undefined`
- Don't hardcode color hex values — use Tailwind theme tokens only
- Don't make the badge a button or link — it is display-only
- Don't show decimal places on the balance number
- Don't add a currency symbol — the coin icon is the only indicator
- Don't animate the number counting up — static display only, no number ticker
