## Why This Exists

Yahya can't read minds. He can't always tell when Yara is in a harder part of her cycle, and he shouldn't have to guess. This card gives him a quiet, non-clinical signal ‚Äî phrased with warmth and humanity ‚Äî so he can show up better without either of them having to say anything uncomfortable. It's not a medical dashboard. It's a gentle nudge in plain language: "Hey, today's a good day to be extra soft with her."

The card lives on the health page and eventually the home dashboard. It changes its color and content automatically based on where Yara is in her cycle. Yahya sees it and just knows. No embarrassment, no guessing. That's the whole point.

This is a YAHYA-ONLY feature. Yara should never see this card ‚Äî not because it's secret, but because it's a tool for him, not for her.

---

## User Story

As Yahya, when I open the health section, I see a card that tells me ‚Äî in plain, kind language ‚Äî what phase Yara is in and how I should think about that today. The card changes color and message depending on whether she's in her active phase, approaching PMS, or in her break/period phase. I don't have to do any math or check any dates. I just read the card and I know.

---

## Full UX Flow

1. **Yahya opens /health** ‚Üí CycleInsightCard renders at the top of the page
2. **Active phase (days 1‚Äì18):** Card has a calm blue-tinted background, Sparkles icon, "Smooth sailing" title, gentle upbeat body copy
3. **PMS window (days 19‚Äì21):** Card shifts to warm amber, Heart icon, "Be extra gentle" title, practical soft guidance
4. **Break/period phase (days 22‚Äì28):** Card is soft rose, Shield icon, "Take care of her" title, actionable care instructions
5. **Yara opens /health** ‚Üí Card returns null, nothing is rendered for her. No trace.
6. **useCycle() returns loading or no data** ‚Üí Card renders nothing (null) rather than showing incorrect or empty guidance

## Phase Logic

Derive the display phase from the values returned by `useCycle()`:

```tsx
const { phase, isPMSWindow, isPeriodLikely } = useCycle()

// Guard: not Yahya ‚Üí return null
// Guard: no data / loading ‚Üí return null

// Priority order:
// 1. isPMSWindow === true  ‚Üí PMS content
// 2. isPeriodLikely === true ‚Üí Break content
// 3. default               ‚Üí Active content
```

The `phase` field from useCycle may also be used directly if it exposes `'active' | 'pms' | 'break'` strings ‚Äî prefer the boolean flags if both are available, as they are more explicit.

## Implementation Notes

- `"use client"` directive required (uses `useCycle` hook which reads Zustand/client state)
- Import `useCycle` from `@/hooks/use-cycle` (T502)
- Import icons from `lucide-react`: `Sparkles`, `Heart`, `Shield`
- Use `cn()` from `@/lib/utils` for conditional class merging on the outer wrapper
- Do not use Tailwind color classes for the phase-specific backgrounds ‚Äî use inline `style` with the hex values or define them as CSS custom properties, since these are semantic health colors not in the default Tailwind palette
- Border color: use `rgba()` or a slightly darkened variant of the bg ‚Äî a utility helper is acceptable if one exists, otherwise inline `style` is fine
- Framer Motion: apply a subtle `initial={{ opacity: 0, y: 6 }}` ‚Üí `animate={{ opacity: 1, y: 0 }}` with `transition={{ duration: 0.3, ease: 'easeOut' }}` on the card so phase changes feel soft, not jarring. Wrap in `AnimatePresence` with a `key` tied to the current phase so the card re-animates when the phase changes.

## Edge Cases

- **useCycle returns null / undefined phase** ‚Üí return `null` from the component, render nothing
- **User is Yara (not Yahya)** ‚Üí return `null` immediately; check the auth user or a `isOwner` flag from `useCycle`
- **Phase transitions** ‚Üí `AnimatePresence` with a `key` prop on the `motion.div` ensures the card animates out and back in when the phase changes, rather than snapping
- **Long body text on small screens (320px)** ‚Üí `leading-relaxed` and full-width card ensure text wraps gracefully without truncation
- **Icon import errors** ‚Üí All three icons (Sparkles, Heart, Shield) are available in `lucide-react`; no additional packages needed

## Do NOT

- Don't fetch data inside this component ‚Äî use `useCycle()` only
- Don't hardcode phase logic with raw day numbers ‚Äî read from the hook's exported flags
- Don't show this card to Yara under any circumstances
- Don't use Tailwind color classes for phase backgrounds ‚Äî the health semantic colors are not in the default palette
- Don't add interactive elements (buttons, links) to this card ‚Äî it is read-only contextual info
- Don't animate the text content separately ‚Äî animate the whole card as a unit
- Don't show a skeleton or loading state ‚Äî silent null is correct when data is unavailable
