## Why This Exists

A love coupon is a small act of care made tangible. It's Yahya telling Yara: "I promise you this ‚Äî just ask." It's Yara giving Yahya something silly, sweet, and redeemable. When either of them opens the CouponWallet, the first thing they see is a list of these cards. The card is the moment of discovery, the moment of remembering, the moment of deciding to redeem.

This is the most "gift-like" component in the entire app. It should feel like holding a physical coupon ‚Äî warm, personal, slightly precious. Not a spreadsheet row. Not a list item. A card you'd want to keep. The perforated left edge is the signature detail ‚Äî it whispers "tear here when ready."

---

## User Story

As Yahya or Yara, when I open my CouponWallet, I see all my coupons laid out as beautiful cards. Each card immediately tells me what the coupon is for, who created it, and whether I can use it right now. The card feels physical ‚Äî it has weight, texture, and warmth. When I tap it, it gives a satisfying micro-press before opening the detail view.

---

## Full UX Flow

1. **CouponWallet renders** ‚Üí maps over coupon array, renders one CouponCard per coupon
2. **User sees the card** ‚Üí reads emoji + title at a glance, category badge grounds the context, status dot tells them if it's usable
3. **Optional thumbnail** ‚Üí if the coupon has an image_url, a small rounded image appears in the top-right corner, making the card feel personalized and rich
4. **User presses the card:**
   - Immediate scale-down feedback (whileTap scale 0.98) ‚Äî tactile, instant
   - On release, card scales back to 1.0
   - `onPress` callback fires with the coupon object ‚Üí caller navigates to CouponDetail (T606)
5. **Compact mode** ‚Üí when `compact={true}`, card reduces vertical padding, hides description, shrinks emoji ‚Äî useful inside summary views or modals
6. **showCreator toggle** ‚Üí when viewing your own sent coupons, showing "From Yahya" is redundant. Caller passes `showCreator={false}` to hide that line.

## Implementation Notes

- `"use client"` directive required ‚Äî uses Framer Motion and event handlers
- Import `Coupon` type from wherever T602 defines it (likely `@/lib/types/coupon.types.ts` or `@/lib/types`)
- Import `useAuth` from T204 to resolve creator display name from a user ID ‚Äî if `coupon.creator_id` matches `auth.user.id`, show "You"; otherwise show the partner's name
- Use `cn()` from `@/lib/utils` for all conditional class merging
- The dashed left border is achieved with Tailwind: `border-l-2 border-dashed border-accent-soft` ‚Äî if `border-accent-soft` is not in the theme, use `border-[#E8D5C0]`
- Framer Motion `motion.div` wraps the entire card with `whileHover`, `whileTap`, and `onClick`
- The `onClick` handler on the motion.div should call `onPress?.(coupon)` ‚Äî always optional, never crash if not provided
- Thumbnail rendering: `{coupon.image_url && <Image src={coupon.image_url} ... />}` ‚Äî use `next/image` with explicit `width={48}` and `height={48}`
- Description truncation: CSS only ‚Äî `line-clamp-2` Tailwind utility (requires `@tailwindcss/line-clamp` or Tailwind v3.3+). In compact mode, render `null` for description entirely.
- In compact mode: reduce padding to `p-3`, emoji to 22px, omit description block entirely
- Status dot is a `span` with `inline-block`, `w-2 h-2`, `rounded-full`, and `mr-1.5` ‚Äî colored via inline style using the status color map

## Edge Cases

- **Very long title** ‚Äî cap at 2 lines with `line-clamp-2`; Playfair Display wraps gracefully at any length
- **No description** ‚Äî if `coupon.description` is null/empty, hide the description block entirely; card should not show a blank gap
- **No image_url** ‚Äî thumbnail slot is simply absent; layout does not leave whitespace on the right
- **Unknown category** ‚Äî fall back to `general` badge styling; never crash on an unexpected string
- **Unknown status** ‚Äî fall back to showing the raw status string with the `text-muted` gray dot; never crash
- **Creator resolution fails** ‚Äî if `useAuth` returns no partner name, fall back to "Partner" as the display string
- **compact + image_url** ‚Äî in compact mode, do not render the thumbnail; keep the compact layout uncluttered
- **onPress not provided** ‚Äî card still renders and animates on hover/tap; just no action fires

## Do NOT

- Don't hardcode color hex values in JSX for anything other than the category badge and status dot ‚Äî use design tokens everywhere else
- Don't use `<img>` directly for the thumbnail ‚Äî use `next/image` for optimization
- Don't use `router.push()` inside this component ‚Äî navigation is the caller's responsibility via `onPress`
- Don't add swipe-to-delete or swipe-to-redeem gestures here ‚Äî those belong in T604 (CouponWallet)
- Don't add a "Redeem" button directly on the card ‚Äî tapping the card opens CouponDetail where redemption happens
- Don't apply glass, blur, or gradient overlay effects to the card background ‚Äî keep it clean and paper-like
- Don't wrap in a `<Link>` ‚Äî the caller decides where tapping leads via `onPress`
- Don't animate the status dot or category badge ‚Äî animation is reserved for the card-level press interaction
