## Why This Exists

The home dashboard is the first thing either user sees when they open Hayah. For Yahya, the cycle is a core part of daily awareness вҖ” knowing what day Yara is on changes how he shows up. Right now the home page has two widget slots: the first holds CoyynsWidget, the second is an empty placeholder. That second slot is wasted space. This task wires the cycle data directly into the home screen so Yahya gets cycle context the moment the app opens вҖ” without navigating anywhere.

This is not a new feature. It is integration work. All the heavy pieces already exist: CycleDayWidget (T503), CycleInsightCard (T506), and the useCycle hook (T502). This task connects them to the surface that matters most.

---

## User Story

As Yahya, when I open the home screen, I see the current cycle day and a brief insight in the second widget slot вҖ” right below the Coyyns widget. I can tap it to go to /health for more detail. I never see raw data; I see a warm, human summary.

As Yara, when I open the home screen, the second slot shows a neutral placeholder. The cycle data is Yahya's view into her cycle, not her own вҖ” she has her own health page for that.

---

## Full UX Flow

1. **Home screen loads** вҶ’ page calls `useCycle()` to get cycle config
2. **If config is non-null (Yahya):**
   - Second widget slot renders `CycleHomeWidget`
   - `CycleHomeWidget` shows `CycleDayWidget` (compact) stacked above `CycleInsightCard`
   - The entire widget is a tappable card with a `next/link` wrapping вҶ’ navigates to `/health`
   - Tap gives scale feedback (Framer Motion `whileTap={{ scale: 0.98 }}`)
3. **If config is null (Yara):**
   - Second widget slot renders a generic `WidgetSlot` placeholder (same as before T508)
   - No empty gap, no layout shift
4. **Navigating to /health from the widget** вҶ’ PageTransition handles the content animation

## Implementation Notes

- `page.tsx` uses the App Router. If it is currently a Server Component, it must become a Client Component (`"use client"`) to call `useCycle`. Check the current file state before deciding.
- If converting `page.tsx` to a client component is undesirable (e.g., it currently uses async data fetching), extract a small `HomeCycleSlot` client component that wraps the conditional, and render that inside the server page instead.
- `useCycle` returns `{ config, currentDay, phase, insight, ... }` вҖ” only `config` is needed here for the ownership check.
- Do not replicate the ownership logic. Trust the hook. `config !== null` is sufficient.
- `CycleHomeWidget` should use `cn()` from `@/lib/utils` for className merging.
- All spacing uses Tailwind tokens вҖ” no hardcoded px values.
- The `Link` wrapper from `next/link` handles navigation. Do not use `useRouter`.

## Edge Cases

- **useCycle returns loading state before config resolves** вҶ’ treat `config === null` as the Yara path; show WidgetSlot. This prevents a flash of empty space on Yahya's screen for a fraction of a second. Acceptable trade-off вҖ” the widget will not appear until config loads.
- **CycleInsightCard has no insight to show** вҶ’ that component handles its own empty state; CycleHomeWidget does not need to know.
- **User navigates to /health and back** вҶ’ the home page re-renders with the same config, no issue.
- **Offline / Supabase unavailable** вҶ’ useCycle should return cached config if available. This task does not change that behavior вҖ” it inherits whatever useCycle does.
- **Screen is narrow (375px)** вҶ’ the two widget slots stack vertically on mobile, so both display full-width. The visual spec above is for wider screens only.
