## Why This Exists

Challenges are one of the most playful and binding rituals in Hayah. When Yahya dares Yara to finish a book before him for 100 CoYYns, that dare needs to live somewhere tangible — not a notification that disappears, not a message that gets buried. The ChallengeCard is that tangible thing. It sits in the marketplace, surfaces in the relationship feed, and holds the full weight of a bet between two people who care about each other.

The card needs to communicate immediately: what's the challenge, what's at stake, who created it, and what state it's in. A challenge that's actively live should feel alive — not dormant. A completed one should feel settled, final. A pending resolution should feel tense, like something is waiting to be decided. Each state carries its own emotional register, and the card must wear that register clearly.

---

## User Story

As Yahya or Yara, when I open the Challenges tab, I see a list of our active and past challenges as cards. Each card tells me exactly what I need to know at a glance: what we bet on, how many CoYYns are at stake, who created it, how much time is left, and what state it's in. If a challenge is active and I'm ready to claim my win, there's a button right there. I don't need to open a detail screen to know what's happening.

---

## Full UX Flow

1. **ChallengeCard renders in a list** — typically inside the CoYYns Marketplace Challenges tab (T701)
2. **User scans the card** → reads title, stakes, and status at a glance
3. **If status is `active`:**
   - A green dot pulses softly in the status row
   - "Claim Win" button is visible if the current user hasn't already claimed
   - "View Details" text button is also visible
4. **User taps "Claim Win"** → `onClaimWin(challenge)` fires → handled by parent (T707 resolution flow)
5. **If status is `pending_resolution`:**
   - Amber dot in status row
   - "Waiting for confirmation..." text replaces buttons — signals tension, one party already claimed
6. **If status is `completed`:**
   - Gray dot in status row
   - "Winner: [name]" replaces buttons — resolved and final
7. **If status is `expired` or `cancelled`:**
   - Gray dot, muted visual treatment, no action buttons
8. **User taps "View Details"** → `onViewDetails(challenge)` fires → parent handles navigation

## Visual Spec

```
┌─────────────────────────────────────┐
│  ⚡ First to finish a book          │  ← emoji + title
│                                     │
│  🪙 100 CoYYns at stake            │  ← stakes in accent-primary
│                                     │
│  Created by Yahya · 5 days left     │  ← creator + deadline
│  Status: Active 🟢                  │  ← status indicator
│                                     │
│  [Claim Win]  [View Details]        │  ← action buttons
└─────────────────────────────────────┘
```

- Card: `bg-elevated` (#FFFFFF), `rounded-2xl`, `shadow-soft`, `p-5`
- Title row: emoji prefix (if present) + title in DM Sans, 15px, `font-semibold`, `text-primary` (#2C2825)
- Stakes row: JetBrains Mono, 13px, `text-accent-primary` (#C4956A) — format: `🪙 {n} CoYYns at stake`
- Meta row: DM Sans, 12px, `text-muted` — "Created by {name} · {n} days left" or "Created by {name} · No deadline"
- Status row: 12px, `text-muted` — colored dot (8px circle) + label
  - `active` → green dot (#7CB67C), "Active"
  - `pending_resolution` → amber dot (#D4A04A), "Pending confirmation"
  - `completed` → gray dot (#8C8279), "Completed"
  - `expired` → gray dot (#8C8279), "Expired"
  - `cancelled` → gray dot (#8C8279), "Cancelled"
- Button row (only for `active` and `pending_resolution`):
  - "Claim Win": small outlined button, `border-accent-primary`, `text-accent-primary`, `rounded-lg`, `px-3 py-1.5`, `text-xs`
  - "View Details": small ghost/text button, `text-secondary`, `text-xs`
- Winner row (only for `completed`): DM Sans, 13px, `text-success` (#7CB67C), "Winner: {name}"
- Waiting row (only for `pending_resolution`): DM Sans, 12px, `text-warning` (#D4A04A), italic — "Waiting for confirmation..."
- Cards that are `expired` or `cancelled` render at reduced opacity: `opacity-60`

## Implementation Notes

- `"use client"` directive required — uses `useAuth` hook and Framer Motion
- Use `useAuth()` from T204 to get `currentUser.id` and to resolve `creator_id` and `winner_id` to display names
  - If `creator_id === currentUser.id` → "Created by You"
  - Else → "Created by Yara" (or the partner's name)
  - Same logic for `winner_id` → "Winner: You" or "Winner: Yara"
- "Claim Win" button visibility: only render if `challenge.status === 'active'`
  - If `onClaimWin` is not provided, the button should not render regardless
- Deadline display: compute from `challenge.deadline` using `formatDistanceToNow` from `date-fns`
  - If deadline is null → display nothing after the creator meta, or show "No deadline"
  - If deadline has passed but status is still `active` → show "Overdue" in warning color
- Use `cn()` from `@/lib/utils` for conditional class merging throughout
- Entry animation: `motion.div` with `initial={{ opacity: 0, y: 8 }}`, `animate={{ opacity: 1, y: 0 }}`, `transition={{ duration: 0.25, ease: 'easeOut' }}`
- Hover: `whileHover={{ y: -1 }}` on the card container, `transition={{ duration: 0.15 }}`
- The status dot should be a small `span` with `rounded-full` and `w-2 h-2` inline with the status label
- For `active` status, wrap the dot in a `motion.div` with `animate={{ scale: [1, 1.3, 1] }}` and `transition={{ repeat: Infinity, duration: 2 }}` for a subtle pulse
- Do not import `date-fns` without confirming it's in `package.json`
- Stakes number: format as integer (no decimals) — `{challenge.stakes.toLocaleString()}`

## Edge Cases

- **Very long title** → clamp to 2 lines with `line-clamp-2`, never overflow the card
- **No emoji** → title renders without any prefix gap or empty space
- **Stakes = 0** → still render the stakes row, show "0 CoYYns at stake"
- **Deadline in the past + status active** → show "Overdue" in `text-warning` color instead of "N days left"
- **winner_id is not null but matches neither known user** → fall back to "Winner: Unknown"
- **`onClaimWin` not provided** → omit the "Claim Win" button entirely, not just disable it
- **`onViewDetails` not provided** → omit the "View Details" button entirely
- **Both buttons absent (completed/expired/cancelled)** → bottom of card ends at status row, no extra gap
- **`pending_resolution` + current user already claimed** → "Claim Win" does not show; only "Waiting for confirmation..." appears

## Do NOT

- Don't hardcode color hex values — use Tailwind theme tokens mapped to CSS variables
- Don't use `router.push()` inside the card — fire callbacks and let the parent navigate
- Don't resolve user names by fetching from Supabase inside this component — use `useAuth()` data only
- Don't add swipe-to-dismiss or swipe gestures — that's not in scope for this card
- Don't add a countdown timer — static deadline text is sufficient at this stage
- Don't truncate the stakes number — always display the full integer
- Don't animate the status dot for any state other than `active`
