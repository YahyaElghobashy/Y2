# T104: LoadingSkeleton — The Art of Waiting

## Read First
- `PROJECT_CONTEXT.md` → `CLAUDE.md` → `docs/DESIGN_SYSTEM.md`

---

## Why This Exists

The period between tapping something and seeing content is where trust is built or broken. A blank screen says "broken." A spinner says "working." A skeleton says "I already know what's coming, and it's going to look like this." Skeletons are placeholders that mirror the shape of real content — they tell the user the layout before the data arrives. In Hayah, skeletons pulse gently in the warm secondary color, feeling like a heartbeat rather than a loading state.

---

## User Story

As a user waiting for content to load (opening a list, fetching data, transitioning to a new view), I see soft, warm-colored shapes that mirror where the real content will appear. The shapes pulse gently — not a harsh shimmer, just a subtle breath. When the real content arrives, it replaces the skeleton seamlessly. I never see a blank white void or a spinning wheel.

## Component API

```tsx
// File: src/components/shared/LoadingSkeleton.tsx

interface LoadingSkeletonProps {
  variant: "card" | "list-item" | "header" | "full-page"
  count?: number  // for list-item: how many rows to render. Default 3.
  className?: string
}

export function LoadingSkeleton({ variant, count = 3, className }: LoadingSkeletonProps) { ... }
```

---

## Variant Specifications

### `card`
Mimics a content card (like QuickActionCard or a coupon card).
- Container: full width, 120px height, `rounded-xl` (12px)
- Background: `bg-secondary` (#F5F0E8)
- CSS `animate-pulse` (Tailwind built-in)
- Inside the card:
  - Top-left: small circle, 32px diameter (icon placeholder)
  - Below circle: rectangle, 60% width, 14px height, rounded (title placeholder)
  - Below title: rectangle, 40% width, 10px height, rounded (subtitle placeholder)
  - All inner elements: slightly lighter than card bg (use `bg-bg-primary` with some opacity)

### `list-item`
Mimics a list row (like transaction history, settings rows, coupon list).
- Renders `count` rows, `gap-3` between them
- Each row: full width, 56px height, `rounded-lg` (8px), `bg-secondary`, `animate-pulse`
- Inside each row (flex, items-center, px-4):
  - Left: 40px circle (avatar/icon placeholder)
  - Right of circle, with gap-3:
    - Top bar: 60% of remaining width, 12px height, rounded
    - Bottom bar: 40% of remaining width, 10px height, rounded
  - Inner elements: `bg-bg-primary` with opacity

### `header`
Mimics a page header area.
- Title bar: 50% width, 28px height, rounded, `bg-secondary`, `animate-pulse`
- Below (8px gap): subtitle bar: 30% width, 16px height, rounded, `bg-secondary`, `animate-pulse`

### `full-page`
Combines header + cards for a complete page loading state.
- Padding: `p-6`
- Header skeleton (using the header variant internally)
- Gap: 24px
- 3 card skeletons stacked with `gap-4` between them

---

## Design Details

- All skeleton blocks: `bg-bg-secondary` (#F5F0E8) — the warm secondary background color
- Inner detail blocks (bars inside cards): `bg-bg-primary` (#FBF8F4) — slightly lighter to create depth
- Animation: Tailwind's `animate-pulse` only. This is a gentle opacity oscillation (1.0 → 0.5 → 1.0, 2s cycle). NOT a shimmer/gradient sweep.
- Border radius matches real components: cards get `rounded-xl`, list items get `rounded-lg`
- NO borders on any skeleton element

---

## Edge Cases

- `list-item` with `count={0}` → render nothing
- `list-item` with `count={1}` → render one row
- Very narrow screen → skeleton bars should use percentage widths, not fixed px
- Used inside a FadeIn wrapper → skeleton fades in, then pulses, then gets replaced by real content

## What This Connects To

- Used in: every module page while data loads (once Supabase is connected in Phase 2+)
- Used in: the full-page variant as a loading.tsx file for Next.js route-level loading
- The `full-page` variant could be used as `src/app/loading.tsx` (Next.js automatic loading UI)
- Before Phase 2 (no data fetching), skeletons exist but aren't actively used — they're ready for when data arrives

---

## Implementation Notes

- This can be a Server Component — no `"use client"` needed (pure CSS animation, no hooks)
- Use `cn()` for class merging
- Each variant is a simple conditional render inside the main component
- Keep it simple — these are just styled divs with pulse animation

## Do NOT

- No shimmer/gradient sweep — pulse only
- No JavaScript-driven animation — CSS only
- No hardcoded colors — theme tokens only
- No spinners anywhere in this component
- No Framer Motion — skeletons use CSS animation only for performance
