# T104: LoadingSkeleton Component

Build loading skeleton placeholder components.

## Read First
- `CLAUDE.md`
- `docs/DESIGN_SYSTEM.md`

## What to Build

File: `src/components/shared/LoadingSkeleton.tsx`
Test: `src/__tests__/components/shared/LoadingSkeleton.test.tsx`

Four skeleton variants that show pulsing placeholders while content loads.

## Component API

```tsx
interface LoadingSkeletonProps {
  variant: "card" | "list-item" | "header" | "full-page"
  count?: number  // for list-item: how many rows. Default 3
  className?: string
}

export function LoadingSkeleton({ variant, count = 3, className }: LoadingSkeletonProps) { ... }
```

## Variants

### card
- Rounded rectangle (rounded-xl, 12px)
- Height: 120px, full width
- Background: `bg-secondary` (#F5F0E8) with CSS `animate-pulse`
- Inside: two short bars (title placeholder 60% width, subtitle 40% width)

### list-item
- Repeats `count` times with small gap between (gap-3)
- Each: 56px height, full width, rounded-lg
- Left: 40px circle (avatar placeholder)
- Right: two lines (60% and 40% width)

### header
- One tall bar: 32px height, 50% width (title)
- One short bar below: 16px height, 30% width (subtitle)
- Gap of 8px between

### full-page
- Combines header + 3 cards with gaps
- Padding: p-6

## Design Specs

- Skeleton blocks: `bg-bg-secondary` (#F5F0E8)
- Animation: CSS `animate-pulse` (Tailwind built-in). NOT a custom shimmer.
- Border radius matches design system (cards 12px, items 8px)
- No borders on skeletons

## Implementation Notes

- Pure CSS animations, no Framer Motion needed for skeletons
- Can be server component (no `"use client"` needed)
- Use `cn()` for class merging

## Test Requirements

- Renders each variant without errors
- list-item variant renders correct count
- Accepts className prop

## Do NOT

- No shimmer/gradient animations — just pulse
- No JavaScript animation — CSS only
- No hardcoded colors — use theme tokens
