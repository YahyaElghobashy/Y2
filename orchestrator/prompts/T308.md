## Why This Exists

The home dashboard has been waiting. Since T108, two WidgetSlot placeholders have held the space that was always meant for live data. This is the task that fills the first one. CoYYns are woven into every day of Yahya and Yara's shared life вҖ” they earn them, spend them, gift them. Making the balance visible the moment you open the app is not a nice-to-have. It's the signal: this app knows you, it remembers what you've done, it's tracking your shared story.

This task also completes the home greeting. Right now "Good morning, Yahya" is hardcoded. After T308, the greeting pulls from the authenticated profile вҖ” so Yara sees her own name. That small detail closes the loop on identity. The home screen finally knows who it's talking to.

---

## User Story

As Yahya or Yara, when I open Hayah, the home screen greets me by my real name and shows me my CoYYns balance at a glance. I can see my last three transactions without navigating anywhere. If I want the full picture, I tap the card and land on the Us page with the CoYYns wallet open. It feels like the app is aware of me вҖ” not a blank template waiting to be filled, but a living record of our shared life.

---

## Full UX Flow

1. **Home page loads** вҶ’ HomeGreeting calls `useAuth()` and resolves `profile.display_name`
2. **Greeting renders** вҶ’ "Good evening, Yara" (or whoever is signed in). If profile hasn't resolved yet, greeting says "Good evening, there" вҖ” never blank, never broken
3. **First widget slot is now CoyynsWidget** вҶ’ compact card with:
   - Top row: label "CoYYns" (left) and CoyynsBadge balance pill (right)
   - Middle section: last 3 transactions in compact list form
   - Bottom row: "See all вҶ’" link to `/us`
4. **If no transactions yet** вҶ’ instead of an empty list, a single line reads "Start earning CoYYns together" вҖ” soft, encouraging, never cold
5. **Tapping the card area** (anywhere except "See all вҶ’") вҶ’ navigates to `/us`
6. **Second WidgetSlot remains** вҶ’ still shows "Widget coming soon" (Phase 5 will replace it with the cycle tracker)
7. **Balance is loading** вҶ’ CoyynsBadge shows its built-in pulse animation. Transaction list shows three LoadingSkeleton rows (list variant). The card shape is present immediately вҖ” no layout shift.

## New Component

### `src/components/home/CoyynsWidget.tsx`

```tsx
"use client"

interface CoyynsWidgetProps {
  className?: string
}

export function CoyynsWidget({ className }: CoyynsWidgetProps) { ... }
```

- Calls `useCoyyns()` from `@/lib/hooks/use-coyyns` вҖ” gets `wallet`, `transactions`, `isLoading`
- Renders a `motion.div` wrapper with `whileTap={{ scale: 0.99 }}` and `transition={{ duration: 0.1 }}` вҖ” light tactile press on the whole card
- The card itself uses `next/link` wrapping the entire card to `/us` вҖ” the "See all вҶ’" text is a visual affordance, not a separate link
- Slices `transactions.slice(0, 3)` вҖ” never shows more than 3
- Shows `LoadingSkeleton` list variant (3 rows) while `isLoading` is true
- Shows empty state text if `!isLoading && transactions.length === 0`
- CoyynsBadge receives no `balance` prop вҖ” it reads from `useCoyyns()` itself (avoid double hook or pass `wallet?.balance` as a prop to prevent two parallel fetches вҖ” prefer passing `wallet?.balance`)
- Amount display: prefix `+` for earn, `-` for spend; render `Math.abs(tx.amount).toLocaleString()`
- Use `cn()` from `@/lib/utils` for conditional classes

## Implementation Notes

- `CoyynsWidget` is a `"use client"` component (calls `useCoyyns()` and uses Framer Motion)
- `HomeGreeting` is already `"use client"` from T108 вҖ” the directive stays, just add `useAuth()`
- `page.tsx` is already `"use client"` from T108 вҖ” no change needed to the directive
- Import `CoyynsWidget` with the path `@/components/home/CoyynsWidget` вҖ” keep all home-specific components co-located in `components/home/`
- CoyynsBadge: pass `balance={wallet?.balance}` to avoid the widget calling `useCoyyns()` twice (once in CoyynsWidget, once inside CoyynsBadge). If `wallet` is null, `balance` prop will be `undefined`, and CoyynsBadge will call the hook itself вҖ” this is acceptable but wasteful. Prefer explicit prop passing.
- Do NOT restructure or refactor CoyynsHistory (T307) вҖ” use it as-is if it supports a `compact` and `limit` prop. If it does not, render the compact 3-row transaction list directly inside CoyynsWidget rather than forcing CoyynsHistory to accept props it wasn't built for.
- Use `date-fns` for any relative time display in transactions if needed (T307 may already handle this)
- The `useCoyyns()` hook (T302) returns `transactions` ordered `DESC` вҖ” `slice(0, 3)` gives the three most recent, which is correct

## Edge Cases

- **Profile not yet loaded** вҶ’ `profile?.display_name` is `undefined` вҶ’ greeting shows "Good morning, there" вҖ” graceful
- **Profile has no display_name set** вҶ’ same fallback as above вҖ” "Good morning, there"
- **Wallet is null** (new user, wallet row not yet created by DB trigger) вҶ’ `wallet?.balance` is `undefined` вҶ’ CoyynsBadge shows loading pulse via its own hook call вҖ” acceptable
- **Transactions is empty array** вҶ’ `transactions.length === 0` вҶ’ show "Start earning CoYYns together" centered in the list area. Do not show LoadingSkeleton for this state.
- **isLoading is true** вҶ’ show LoadingSkeleton list variant for transactions, CoyynsBadge handles its own loading state
- **Transactions has exactly 1 or 2 entries** вҶ’ `slice(0, 3)` returns fewer rows вҖ” render however many exist, do not pad with empty placeholders
- **Long description text** вҶ’ truncate with `truncate` class and `max-w-[160px]` вҖ” never wrap to a second line, preserves single-row transaction layout
- **Amount is 0** вҶ’ unlikely given T302 validation, but render "0" вҖ” not blank
- **Auth hook errors** вҶ’ `useAuth()` returns null profile, fallback "there" applies. No error state is shown in the greeting вҖ” silent degradation only.
- **navigation to /us**: use `next/link` (not `router.push`) for the card wrapper link
