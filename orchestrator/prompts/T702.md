## Why This Exists

The CoYYns marketplace is where earned currency becomes meaning. Yahya accumulates CoYYns by logging workouts, completing check-ins, keeping streaks alive. Yara earns them through her own rituals. When either of them opens the marketplace, they're looking at what they can unlock — an extra notification send, a surprise voucher, a date night fund contribution. The MarketplaceItemCard is the unit of desire in that browsing experience.

Each card must communicate three things instantly: what it is, what it costs, and whether it's within reach. If the balance is short, the card should feel honest about it — muted, but not punishing. If the item is coming soon, it should feel like a preview, not a dead end.

---

## User Story

As Yahya or Yara, when I browse the CoYYns marketplace, I see a grid of item cards. Each card shows me the item's icon, what it does, and how many CoYYns it costs. If I can afford it, the price badge glows warm copper. If I can't, it's muted and the card fades slightly so I know to keep earning. If I tap a card I can afford, it triggers the purchase flow.

---

## Full UX Flow

1. **Marketplace page loads** → grid of MarketplaceItemCards appears
2. **Each card shows** icon, title, short description, and price badge
3. **User scans cards** — affordable items are visually prominent, unaffordable items are faded
4. **User hovers (desktop) or long-presses (mobile)** → card scales up slightly (1.02)
5. **User taps an affordable card** → scale down (0.98) on press, then `onPurchase()` fires on release
6. **User taps an unaffordable card** → no interaction, no callback, no visual press response
7. **Coming soon card** → no price badge, fully muted, not tappable, shows "Coming soon" label instead of price

- Card: `bg-elevated` (#FFFFFF), `rounded-2xl`, shadow-soft, `p-5`, full-width in grid cell
- Icon: ReactNode, 32px container, rendered top-left before title
- Title: DM Sans, 15px (`text-[15px]`), `font-semibold`, `text-primary` (#2C2825)
- Description: DM Sans, 13px (`text-[13px]`), `text-secondary` (#8C8279), `line-clamp-2`
- Price badge: inline-flex, `rounded-full`, `px-3 py-1`, `bg-accent-soft` (#E8D5C0), `text-accent-primary` (#C4956A)
  - Coin emoji + number in JetBrains Mono + "CoYYns" label in DM Sans
- Affordable state: full opacity, normal colors, cursor-pointer, hover/tap animations active
- Can't afford state: `opacity-70`, price badge uses `bg-muted/30` and `text-muted` (#B5ADA4), `cursor-not-allowed`, no animations
- Coming soon state: no price badge, show small "Coming soon" pill in `text-muted`, `opacity-60`, `cursor-default`, no animations
- Hover: `whileHover={{ scale: 1.02 }}` (only when affordable and available)
- Tap: `whileTap={{ scale: 0.98 }}` (only when affordable and available)
- Transition: `{ type: "spring", stiffness: 400, damping: 25 }`

## Implementation Notes

- `"use client"` directive required (uses Framer Motion)
- Use `motion.div` as the card wrapper to enable `whileHover` and `whileTap`
- Only apply `whileHover` and `whileTap` when `available && affordable` — guard with a conditional props spread
- The `onPurchase` callback fires only when `available && affordable` — add a guard inside `onClick`
- Use `cn()` from `@/lib/utils` for all conditional class merging
- Icon wrapper: a plain `div` sized to 32px, no background needed — the icon renders as-is
- Price number must use `font-mono` (JetBrains Mono) via `font-['JetBrains_Mono']` or the `font-mono` Tailwind alias
- "CoYYns" label next to the number uses DM Sans (body default), slightly smaller is fine
- The coin emoji (🪙) sits before the number inside the badge — it is a literal emoji in JSX, not a Lucide icon
- No internal state — this is a fully controlled, pure presentational component
- Do not import or call `useCoyyns` inside this component — affordability is passed as a prop from the parent

---

## Edge Cases

- **Long title** → should not overflow; use `line-clamp-1` or allow wrap to 2 lines max, keep card height consistent in a grid
- **Long description** → `line-clamp-2` enforced, overflow hidden
- **Price of 0** → should not happen in practice, but if `price === 0` render "Free" in the badge instead of "0 CoYYns"
- **No `onPurchase` prop** → card renders normally but tapping does nothing; no error thrown
- **Icon is null or undefined** → render a fallback 32px empty placeholder `div` to preserve layout
- **Both `available: false` and `affordable: false`** → `available: false` takes priority — show "Coming soon" state
- **Grid reflow** → card must not jump size when toggling between affordable/unaffordable states

## Do NOT

- Don't call `useCoyyns()` inside this component — affordability is a prop
- Don't hardcode any color hex values — use Tailwind design tokens only
- Don't add a loading/spinner state — that belongs in the purchase modal (T703)
- Don't add quantity selectors or input fields — single tap to purchase
- Don't use `left`/`right` CSS properties — use logical properties (`ps`, `pe`) for RTL readiness
- Don't animate when `available: false` or `affordable: false`
- Don't show an error message inside the card if unaffordable — just visual dimming
