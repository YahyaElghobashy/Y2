## Why This Exists

CoYYns is the shared micro-currency of Yahya and Yara's relationship — earned through acts of care, spent on meaningful rewards. The history list is where that story is told. When Yara checks what she's earned this week, or Yahya sees he's getting close to his coupon redemption goal, this component is what they read. Every row is a moment in their relationship, not just a transaction. The list needs to feel light and legible — like a warm journal entry, not a bank statement. Numbers in mono. Acts of love in prose.

---

## User Story

As Yahya or Yara, I want to see a scrollable list of my CoYYns activity so I can understand what I've earned, what I've spent, and how we're engaging with our shared life. Each row should tell me immediately: was this a gain or a loss, what was it for, and when did it happen — all at a glance without effort.

---

## Full UX Flow

1. **Component mounts** → fetches transactions via `useCoyyns()` hook (T302), or accepts them as a prop
2. **Transactions load** → list animates in using StaggerList (T103) — rows enter with a staggered fade + translateY from below
3. **Each row shows:**
   - A directional icon left-aligned: `TrendingUp` (Lucide) in success green for earn, `TrendingDown` in error red/copper for spend
   - Description text (truncated if too long) to the right of the icon
   - Amount right-aligned in JetBrains Mono: `+50` in success color for earn, `−25` in error color for spend
   - Below the description: category tag + " · " + relative time in muted text
4. **Empty state** → if no transactions exist, render `EmptyState` component (T105) with a coin icon, "No CoYYns yet" title, and "Acts of care will appear here" subtitle
5. **Limit prop** → only the first `limit` items are rendered (default 50); no pagination needed at this stage
6. **Compact mode** → reduces row padding and font size slightly for embedding inside cards or summary views

## Visual Spec

```
┌──────────────────────────────────────────────┐
│  ↗  Helped with groceries            +50     │  ← earn row
│     manual · 2 hours ago                     │
├──────────────────────────────────────────────┤
│  ↙  Extra notification               −25     │  ← spend row
│     notification_purchase · yesterday         │
├──────────────────────────────────────────────┤
│  ↗  Cooked dinner together           +30     │
│     manual · 3 days ago                      │
└──────────────────────────────────────────────┘
```

**Row anatomy:**
- Row height: 56px default, 44px in compact mode
- Left icon container: 32px × 32px, rounded-lg, soft tinted background (success-tint or error-tint at ~10% opacity), icon size 16px
- Description: 14px, `font-body` (DM Sans), `text-primary` (#2C2825), truncate with ellipsis at single line
- Category + time: 12px, `text-muted` (#B5ADA4), separator " · " between them
- Amount: 14px, `font-mono` (JetBrains Mono), font-semibold, right-aligned, "+" prefix for earn (success #7CB67C), "−" prefix for spend (error #C27070)
- Row separator: 1px border in `border-subtle` (rgba(44,40,37,0.06)), no separator after last row
- Row background: `bg-elevated` (#FFFFFF) with subtle hover state on pointer devices

**List container:**
- Background: transparent (inherits page bg)
- Overflow: scroll on y-axis when embedded; natural document flow when standalone
- No horizontal scroll
- Padding: 0 (rows handle their own internal padding)

**CoyynsTransaction shape** (defined in T302):
```ts
interface CoyynsTransaction {
  id: string
  amount: number         // positive = earn, negative = spend
  type: 'earn' | 'spend'
  category: string       // e.g. "manual", "notification_purchase"
  description: string
  created_at: string     // ISO timestamp
}
```

## Implementation Notes

- `"use client"` directive required (uses hooks and Framer Motion)
- Import `useCoyyns` from `@/hooks/use-coyyns` (T302) — only call the hook when `transactions` prop is not provided; use the prop directly when it is
- Use `TrendingUp` and `TrendingDown` from `lucide-react` for the earn/spend icons
- Relative time: use a simple helper or `date-fns/formatDistanceToNow` — check `package.json` first; do NOT install a new package if one already exists
- StaggerList (T103) wraps the rows — each `<li>` is a stagger child, entrance animation is `fadeUp` (opacity 0→1, translateY 8px→0), stagger delay 40ms per item
- EmptyState (T105) renders when the resolved transaction list is empty (after applying limit)
- For the icon background tint: use inline style with `rgba(124,182,124,0.10)` for earn, `rgba(194,112,112,0.10)` for spend — these are one-off values acceptable as inline styles since they derive from the semantic colors
- Amount display: `Math.abs(amount)` for the number; prefix "+" if `type === 'earn'`, "−" (U+2212 minus sign, not hyphen) if `type === 'spend'`
- Apply `limit` by slicing: `transactions.slice(0, limit)` before rendering
- Use `cn()` from `@/lib/utils` for all conditional class merging
- Compact mode: row `py-2` instead of `py-3`, description 13px instead of 14px, category+time hidden if needed to conserve space

## Edge Cases

- **Single transaction** → list renders with one row, no visual glitch, StaggerList handles single child gracefully
- **Very long description** → truncate at one line with ellipsis; do not wrap or push amount off screen
- **Large amounts** (e.g. +1000) → JetBrains Mono handles wide numbers; amount column must not overflow its container
- **Unknown category** → render the raw category string as-is; no crash, no placeholder needed
- **Loading state** → if `useCoyyns()` is still loading, render a subtle skeleton (3 rows of gray bars) rather than a blank space — keep it consistent with LoadingSkeleton (T104) patterns
- **Error from hook** → fail silently with EmptyState rather than crashing; log to console in dev mode
- **Compact + empty** → EmptyState still renders in compact mode; it should respect the compact sizing context

## Do NOT

- Don't hardcode hex color values — use Tailwind theme tokens or CSS variables
- Don't render a table element — use a styled list (`ul` / `li`) for correct semantics and animation compatibility
- Don't add sorting or filtering UI — that is a future feature
- Don't show a "load more" button — limit is a hard cap for now
- Don't use a hyphen `-` as the minus sign — use U+2212 `−` for spend amounts
- Don't install `date-fns` or any time library if one is already in `package.json` — check first
- Don't skip the loading skeleton — a blank flash before data loads is a poor experience
