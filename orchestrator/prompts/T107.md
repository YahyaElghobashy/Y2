# T107: AppShell Layout

Build the app shell that wraps all authenticated pages.

## Read First
- `CLAUDE.md`
- `docs/DESIGN_SYSTEM.md`
- `docs/ARCHITECTURE.md`

## What to Build

File: `src/components/shared/AppShell.tsx`
Test: `src/__tests__/components/shared/AppShell.test.tsx`

The main layout wrapper. Every page except `/login` renders inside this. It provides the BottomNav and manages the scroll container.

## Component API

```tsx
interface AppShellProps {
  children: React.ReactNode
}

export function AppShell({ children }: AppShellProps) { ... }
```

## Structure

```
┌──────────────────────────┐
│  (page content scrolls)  │
│                          │
│   children go here       │
│                          │
│                          │
│                          │
│                          │
├──────────────────────────┤
│  BottomNav (fixed)       │
│  ──────────────────────  │
│  safe area padding       │
└──────────────────────────┘
```

## Design Specs

- Full screen: `min-h-screen` (or `min-h-[100dvh]` for mobile viewport units)
- Background: `bg-primary` (#FBF8F4)
- Content area: scrollable, with `pb-20` to account for BottomNav height
- BottomNav: fixed at bottom (`fixed bottom-0 left-0 right-0 z-50`)
- Safe area: the content and nav both respect `env(safe-area-inset-bottom)` for iPhone home bar

## Implementation Notes

- `"use client"` (imports BottomNav which uses hooks)
- Import `BottomNav` from `@/components/shared/BottomNav`
- The content area should have padding: `px-0` (individual pages add their own horizontal padding — this allows full-bleed cards)
- Vertical padding: `pt-2` at top (pages add their own PageHeader), `pb-24` at bottom (clears nav + some breathing room)

## Integration

After building this, update `src/app/layout.tsx` to wrap `{children}` in `<AppShell>`:

```tsx
// In layout.tsx body:
<body className="...">
  <AppShell>
    {children}
  </AppShell>
</body>
```

**IMPORTANT:** Only do this wrapping if the auth system is not yet in place. If auth middleware exists, the shell should only wrap authenticated routes. For now (Phase 1), wrap everything.

## Test Requirements

- Renders children
- BottomNav is present in the output
- Content area has bottom padding

## Do NOT

- Don't add a top app bar / status bar — PageHeader handles that per-page
- Don't add scroll-to-top behavior
- Don't add a backdrop blur on the nav
