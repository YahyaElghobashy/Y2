## Why This Exists

CoYYns are how Yahya and Yara recognise effort and thoughtfulness in their shared life — cleaning the kitchen, planning a date, making breakfast when the other one is tired. They are earned intentionally, not automatically, and that matters. The AddCoyynsForm is the ritual moment of that intention: you open it, you say what you did, you log the value. The form must feel deliberate and warm, not like a spreadsheet. Playfair Display for the title, monospace for the number, copper for the action button. This is a small ceremony, not a data entry screen.

---

## User Story

As Yahya or Yara, when I tap the "+" button on the CoyynsWallet screen, a bottom sheet slides up from the bottom. I type how many CoYYns I earned and a short note about what I did to earn them. I tap "Add" and the sheet closes with a brief toast confirming the addition. My CoYYns balance updates on the wallet screen without a page reload.

---

## Full UX Flow

1. **CoyynsWallet (T303)** triggers `onAdd` → sets parent `open` state to `true` → passes `open={true}` to this component
2. **Sheet slides up** from the bottom of the viewport — ease-out, 300ms, translates from `y: "100%"` to `y: 0`
3. **Backdrop** dims the content behind with `bg-black/30`, tapping it calls `onClose`
4. **Form is visible:**
   - Title: "Add CoYYns" in Playfair Display, 22px, `text-primary`
   - Amount input: large centered number field in JetBrains Mono, placeholder "0", copper underline on focus
   - Description textarea: smaller, placeholder "What did you earn this for?", 3 rows
   - "Add" button: copper (`bg-accent-primary`), full width, rounded-xl
   - X button: top-right corner, `text-secondary`, closes the sheet
5. **User submits:**
   - React Hook Form validates via Zod schema
   - If invalid → inline error messages appear below the relevant field, form does not close
   - If valid → calls `addCoyyns({ amount, description })` from `useCoyyns` hook
6. **On success:**
   - Sheet closes (slides back down)
   - `onSuccess?.()` is called so CoyynsWallet can refetch or optimistically update
   - A brief toast appears: "CoYYns added!" (use shadcn `toast` or a simple positioned div — whichever is already in the project)
7. **On error:**
   - Sheet stays open
   - Inline error appears below the button: "Something went wrong. Try again."
8. **Swipe down to dismiss** — if the user swipes down on the sheet, call `onClose` (optional drag gesture via Framer Motion `drag="y"` and `dragConstraints`)

## Form Schema

```tsx
import { z } from "zod"

const addCoyynsSchema = z.object({
  amount: z
    .number({ invalid_type_error: "Enter a number" })
    .int("Must be a whole number")
    .min(1, "Must be at least 1")
    .max(10000, "Maximum 10,000"),
  description: z
    .string()
    .min(1, "Tell us what you earned it for")
    .max(200, "Keep it under 200 characters"),
})

type AddCoyynsFormValues = z.infer<typeof addCoyynsSchema>
```

## Implementation Notes

- `"use client"` directive required (uses React Hook Form, Framer Motion, and hooks)
- Use `useForm<AddCoyynsFormValues>` from `react-hook-form` with `zodResolver(addCoyynsSchema)`
- Register `amount` with `valueAsNumber: true` so the string input is coerced to a number before Zod validation
- Render the sheet using Framer Motion `AnimatePresence` + a `motion.div` — do NOT use shadcn Sheet/Dialog for the outer shell; this is a custom bottom sheet to control the slide animation precisely
- The backdrop is a separate `motion.div` with `initial={{ opacity: 0 }}` and `animate={{ opacity: 1 }}`
- The sheet panel: `initial={{ y: "100%" }}` → `animate={{ y: 0 }}` → `exit={{ y: "100%" }}`, `transition={{ duration: 0.3, ease: [0.25, 0.46, 0.45, 0.94] }}`
- Call `useCoyyns()` from `@/hooks/use-coyyns` (T302) to get the `addCoyyns` mutation function
- After successful submission call `reset()` on the form so the sheet opens fresh next time
- Use `cn()` from `@/lib/utils` for conditional class names
- Safe area: outer sheet wrapper gets `pb-[env(safe-area-inset-bottom)]`
- Portal: render the backdrop + sheet via a React portal to `document.body` so z-index stacking works correctly with fixed positioned BottomNav

## Edge Cases

- **User opens sheet, types nothing, taps Add** → Zod errors appear, form does not submit
- **Amount field accepts only digits** → use `inputMode="numeric"` on the input; Zod `valueAsNumber` handles the conversion
- **User types 10001** → error "Maximum 10,000" appears immediately on blur or on submit
- **Description reaches 200 characters** → show character count (e.g., "180/200") below the textarea so user knows they're near the limit
- **addCoyyns throws an error** → sheet stays open, show inline error below the button, do not call onClose or onSuccess
- **User rapidly opens and closes** → AnimatePresence handles mount/unmount; ensure form is reset on close so stale values don't persist
- **Sheet open on small screen (iPhone SE, 320px)** → amount input and button must still be fully visible without scrolling; reduce font-size of amount to `text-4xl` if needed via responsive classes

## Do NOT

- Don't use shadcn Sheet or Dialog as the outer container — the slide-up animation must be custom Framer Motion
- Don't hardcode `#C4956A` or any hex values — use Tailwind tokens from the theme
- Don't use `router.push()` or any navigation inside this form
- Don't persist form state between opens — always `reset()` after close
- Don't add optimistic UI for the balance update inside this component — that is CoyynsWallet's responsibility via `onSuccess`
- Don't block the submit button while loading with only a disabled state — also show a subtle spinner or loading text so the user knows it's working
