## Why This Exists

Settings is where the app becomes personal. Profile info, notification preferences, appearance options, and the logout button. For Phase 1, most of it is placeholder — but the structure matters. A settings page that feels organized and polished (even with non-functional rows) communicates that the app is mature and well-built. It's also where the "Made with ❤️ for Yara" lives — a small easter egg that makes this feel handcrafted, not mass-produced.

---

## User Story

As Yahya or Yara, when I go to Settings, I see my profile card at the top (avatar, name, email), followed by organized sections: Account, Appearance, About. Each row is tappable and clear. At the bottom, a red "Log Out" button. The page feels clean and structured — like a well-organized drawer. The "Made with ❤️ for Yara" at the bottom makes me smile.

---

## Full UX Flow

1. **Navigate to /settings** (from home — settings gear icon will be added to home later; for now, accessible by typing URL or adding a settings link)
2. **PageHeader:** "Settings" with back arrow to `/`
3. **Profile card** at top: avatar circle with initial, display name, email
4. **Section: Account**
   - Profile → links nowhere yet
   - Notifications → links nowhere yet
5. **Section: Appearance**
   - Theme → shows "Light" on the right side (no toggle, placeholder)
6. **Section: About**
   - App Version → shows "1.0.0" on the right
   - Made with ❤️ → shows "for Yara" on the right
7. **Log Out button** at the bottom — red text, centered, tappable but no-op for now
8. Everything wrapped in PageTransition

## Files to Build

### `src/components/shared/SettingsRow.tsx`

A reusable row component for settings lists.

```tsx
"use client"

interface SettingsRowProps {
  icon: React.ReactNode          // Lucide icon
  label: string                  // "Profile", "Notifications", etc.
  subtitle?: string              // optional secondary line
  href?: string                  // if tappable link
  onClick?: () => void           // if tappable action
  rightElement?: React.ReactNode // toggle, value text, badge
  destructive?: boolean          // red styling for logout
  showChevron?: boolean          // show right arrow (default: true if href/onClick, false otherwise)
}

export function SettingsRow({ ... }: SettingsRowProps) { ... }
```

**Visual:**
- Height: auto, min 56px, `px-6 py-3.5`
- Layout: flex row, items-center
- Left: icon (20px), `text-secondary` (or `text-red-500` if destructive), `mr-3`
- Center: label column
  - Label: DM Sans, 15px, `text-primary` (or `text-red-500` if destructive)
  - Subtitle (if present): 12px, `text-secondary`, `mt-0.5`
- Right: either `rightElement` content OR a `ChevronRight` icon (16px, `text-muted`)
- `showChevron` defaults to true when `href` or `onClick` exists, false when only `rightElement`
- Separator: `border-b border-border-subtle` on all rows except the last in a group (let the parent handle this with group styling)
- If `href`: wrap in `Link` from next/link
- If `onClick`: wrap in `button` with full width
- Press state: subtle background highlight `active:bg-bg-secondary` transition
