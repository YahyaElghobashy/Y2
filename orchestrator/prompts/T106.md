# T106: QuickActionCard — Doorways to Modules

## Read First
- `PROJECT_CONTEXT.md` → `CLAUDE.md` → `docs/DESIGN_SYSTEM.md`

---

## Why This Exists

The home dashboard is the lobby of the app. The QuickActionCards are the four doors leading to each module. When Yahya opens Hayah, his eyes land on these cards and his thumb reaches for whichever one matches his intent. "I want to check on Yara's cycle" → Health card. "I want to send her a notification" → Us card. These cards need to be inviting, clear, and tactile. They're not just links — they're invitations.

---

## User Story

As a user on the home dashboard, I see four beautiful cards arranged in a 2×2 grid. Each card represents a module of our shared life — Us, Health, Spirit, Ops. Each has a distinctive icon in a warm circle, a bold label, and a brief description. When I press the card, it responds physically — pushing down under my thumb. When I release, it navigates to that module. The cards feel like real objects, not flat rectangles.

---

## Full UX Flow

1. Home dashboard loads → four cards appear in a 2×2 grid below the greeting
2. Each card shows: a warm icon circle (top-left), a bold label below it, a one-line description below that
3. Pressing a card: scale(0.98) — the card pushes down (100ms)
4. Releasing: navigates to the module route
5. On desktop hover: scale(1.02) + shadow deepens (150ms ease-out)
6. Navigation triggers → PageTransition on the target page handles the entrance animation

## Visual Spec

- Container: `bg-elevated` (#FFFFFF), `rounded-xl` (12px), padding `p-5`
- Shadow: `shadow-soft` (0 2px 12px rgba(44,40,37,0.06))
- Width: controlled by parent grid (will be in a `grid grid-cols-2 gap-4` on the home page)
- Icon circle: 40px × 40px, `rounded-full`, `bg-accent-soft` (#E8D5C0), icon inside is 20px `text-accent-primary` (#C4956A), centered
- Label: DM Sans, 15px, `font-semibold`, `text-primary`, `mt-3`
- Description: DM Sans, 12px, `text-secondary`, `mt-0.5`, single line, `truncate` if overflows
- Hover animation: `whileHover={{ scale: 1.02 }}` with shadow expansion
- Tap animation: `whileTap={{ scale: 0.98 }}`
- Transition: `{ duration: 0.15, ease: [0.25, 0.1, 0.25, 1] }`
- The entire card is wrapped in a `next/link` Link, with the `motion.div` as the visual container inside it

---

## Edge Cases

- Very long description → truncate with ellipsis (single line)
- Very long label → should never happen (labels are one word), but truncate just in case
- Small screen (320px) → 2-column grid still works, cards just get narrower. Padding adjusts via responsive Tailwind if needed.
- No hover on mobile → `whileHover` is ignored on touch devices, only `whileTap` fires

## Do NOT

- Don't add a border — shadow only, never borders on cards
- Don't add an arrow/chevron — the whole card is the tap target
- Don't make the card tall — it should be compact, ~110px total height
- Don't add notification badges or live data
