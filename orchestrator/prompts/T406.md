## Why This Exists

Every day, each user gets 2 free notification sends to their partner. That limit exists to keep sends feeling intentional and special — not noise. But the user needs to know at a glance how many sends they have left before they commit to sending something. This component lives inside the NotificationBuilder and answers one question instantly: "Can I still send something today?"

The indicator is small and compact — it doesn't dominate the UI. It sits quietly near the send button and tells the truth. When sends run out, it doesn't just go red and disappear — it offers a way forward via the "Buy more" link that opens the BuyExtraSend flow.

---

## User Story

As Yahya or Yara, when I open the notification builder to send my partner a message, I can immediately see how many sends I have left today. If I have sends left, I feel confident to proceed. If I only have one left, I know to make it count. If I have none, I know I need to buy more before I can send — and I can do that with a single tap.

---

## Full UX Flow

1. **User opens NotificationBuilder** → SendLimitIndicator is visible near the send button
2. **2 sends remaining** → two filled dots + "2 sends left today" in success green — user proceeds freely
3. **1 send remaining** → one filled dot, one empty dot + "1 send left today" in warning amber — user is aware this is their last free send
4. **0 sends remaining** → two empty dots + "No sends left · Buy more" in error red — the "Buy more" text is tappable, calls `onBuyMore` callback which opens BuyExtraSend sheet
5. **Bonus sends active** → shows base remaining first, then bonus count separately: "1 send left (+ 2 bonus)"
6. **User buys extra sends** → parent updates `remainingSends` / `bonusSends` props, indicator re-renders with new state

## Visual Spec

```
●● 2 sends left today          ← success green  (#7CB67C)
●○ 1 send left today           ← warning amber  (#D4A04A)
○○ No sends left · Buy more    ← error red      (#C27070) + tappable link

```

- Layout: `inline-flex`, `items-center`, `gap-2`
- Dots container: `flex items-center gap-1`
- Each dot: 8px × 8px circle (`w-2 h-2 rounded-full`)
  - Filled: `bg-[status-color]` (success / warning / error based on remaining count)
  - Empty: `border border-[status-color] bg-transparent`
- Text: DM Sans, 13px (`text-[13px]`), `text-secondary` (#8C8279) for the label
- "Buy more": same 13px, `text-accent-primary` (#C4956A), `cursor-pointer`, no underline by default, hover underline
- Separator dot between "No sends left" and "Buy more": `·` character, `text-muted` (#B5ADA4)
- Bonus note `(+ N bonus)`: same 13px, `text-muted` (#B5ADA4), appended inline

## Implementation Notes

- Pure presentational component — no hooks, no data fetching, no side effects
- The color state is derived from `remainingSends`:
  - `>= 2` → success (`#7CB67C`)
  - `=== 1` → warning (`#D4A04A`)
  - `<= 0` → error (`#C27070`)
- The two dots are always rendered — their filled/empty state reflects how many sends remain (capped at 2 dots regardless of bonus sends)
- Bonus sends are purely additive text — they do not add more dots
- `onBuyMore` is optional; if not provided, the "Buy more" link still renders when `remainingSends <= 0` but clicking it is a no-op (use a defensive check before calling)
- Use `cn()` from `@/lib/utils` for all conditional class merging
- No Framer Motion needed — this component is static; color changes happen via prop-driven re-renders
- Do NOT use hardcoded hex values in className — use inline style or CSS variables where Tailwind tokens are not available for these specific status colors

---

## Edge Cases

- **`remainingSends` is negative** → treat as 0 (show error state, two empty dots, "Buy more")
- **`remainingSends` > 2** → still show two filled dots (the dots cap at 2; the text shows the actual number)
- **`bonusSends` is 0 or undefined** → do not render the bonus parenthetical at all
- **`bonusSends` is negative** → treat as 0, suppress bonus note
- **`onBuyMore` is undefined and remainingSends <= 0** → "Buy more" still renders as a visual affordance, just does nothing on click
- **Very long bonus number** → text wraps naturally; no truncation needed at this scale

## Do NOT

- Don't call any hooks inside this component — it receives all data as props
- Don't hardcode hex color values as Tailwind class strings — use CSS variables or inline style for status colors
- Don't add animation — no Framer Motion, no transitions on dot color changes
- Don't render more than 2 dots regardless of `remainingSends` or `bonusSends` value
- Don't add an underline to "Buy more" by default — only on hover
- Don't make this component async or add any loading state — it is always ready when its props are defined
