## Why This Exists

The /health page has been a warm placeholder since T110. It was built that way intentionally — both Yahya and Yara land on this page and it had to be safe for both of them before the cycle components existed. Now that all four cycle components are built (CycleDayWidget, CycleConfigForm, CycleCalendarView, CycleInsightCard), it's time to wire them in.

This page wiring is the moment where all the cycle work becomes real for Yahya. He opens the health screen and sees the full picture — what day it is, what phase she's in, the month laid out in color, and a quiet card telling him how to show up today. One tap on the gear icon and he can update the settings if anything changes.

For Yara, the page stays exactly as it was in T110 — a clean, warm empty state about general wellness. She sees no gear icon, no cycle components, no configuration UI, and nothing that hints at the cycle system. This is non-negotiable. The split must be clean, hard, and permanent.

---

## User Story

As Yahya, when I open the Health page, I see my full cycle dashboard: the current day and phase widget at the top, a contextual insight card telling me how to be present today, and a calendar view of the full month color-coded by phase. A gear icon in the page header opens cycle settings. If I have not set up tracking yet, I see a friendly prompt to do so.

As Yara, when I open the Health page, I see exactly what I always saw — a warm empty state about wellness features. Nothing about cycles. No gear icon. No hint of the system that exists for Yahya.

## Full UX Flow

### Yahya's Flow (cycle owner)

1. Yahya taps "Health" in BottomNav → page loads with PageTransition
2. **PageHeader:** "Health" with back arrow to `/` and a gear icon (`Settings` from Lucide) on the right slot
3. **`useCycle()` is called** — the hook determines if the user is the cycle owner internally
4. **If `cycleConfig` is null** (no setup yet):
   - Render the EmptyState variant: icon = `Activity`, title = "Set up cycle tracking", subtitle = "Configure your cycle schedule to unlock daily phase insights and the calendar view"
   - EmptyState has a `ctaLabel` = "Get started" that taps to open CycleConfigForm (set `configOpen` to true)
   - Gear icon in PageHeader also opens CycleConfigForm
5. **If `cycleConfig` exists:**
   - Render the full dashboard in a vertical stack with `gap-6`:
     1. `CycleDayWidget` — `compact={false}` — current day and phase at the top
     2. `CycleInsightCard` — contextual soft guidance below the widget
     3. `CycleCalendarView` — full month view at the bottom
   - Gear icon in PageHeader opens CycleConfigForm in edit mode (passes `initialConfig={cycleConfig}`)
6. **CycleConfigForm** is always mounted in the tree (controlled by `configOpen` state) so the bottom sheet animates in smoothly. It receives `open={configOpen}`, `onClose={() => setConfigOpen(false)}`, and `initialConfig={cycleConfig}` (null on first setup).

### Yara's Flow (not cycle owner)

1. Yara taps "Health" in BottomNav → page loads with PageTransition
2. **PageHeader:** "Health" with back arrow to `/` — no gear icon, no right action slot
3. **`useCycle()` `isOwner` is false** — the Yara branch renders
4. EmptyState: icon = `Activity`, title = "Your wellness, tracked", subtitle = "Fitness goals, health reminders, and wellness insights — all in one place" (same as T110)
5. No `CycleConfigForm` in the tree. No cycle-related markup. Nothing.

## File to Modify

### `src/app/health/page.tsx`

This is the only file that changes. All four cycle components (T503–T506) are imported and composed here. The page becomes a `"use client"` component because it manages `configOpen` state and calls `useCycle()`.

```tsx
"use client"

import { useState } from "react"
import { Settings, Activity } from "lucide-react"
import { PageHeader } from "@/components/shared/PageHeader"
import { PageTransition } from "@/components/animations"
import { EmptyState } from "@/components/shared/EmptyState"
import { CycleDayWidget } from "@/components/health/CycleDayWidget"
import { CycleInsightCard } from "@/components/health/CycleInsightCard"
import { CycleCalendarView } from "@/components/health/CycleCalendarView"
import { CycleConfigForm } from "@/components/health/CycleConfigForm"
import { useCycle } from "@/hooks/use-cycle"

export default function HealthPage() {
  const [configOpen, setConfigOpen] = useState(false)
  const { isOwner, cycleConfig, isLoading } = useCycle()

  if (isLoading) return null  // silent — never show a flash of Yara's view to Yahya

  // --- Yara's branch ---
  if (!isOwner) {
    return (
      <PageTransition>
        <PageHeader title="Health" backHref="/" />
        <div className="px-6 py-8">
          <EmptyState
            icon={<Activity size={48} />}
            title="Your wellness, tracked"
            subtitle="Fitness goals, health reminders, and wellness insights — all in one place"
          />
        </div>
      </PageTransition>
    )
  }

  // --- Yahya's branch ---
  return (
    <PageTransition>
      <PageHeader
        title="Health"
        backHref="/"
        rightAction={
          <button
            onClick={() => setConfigOpen(true)}
            className="p-2 text-secondary hover:text-primary transition-colors"
            aria-label="Cycle settings"
          >
            <Settings size={20} />
          </button>
        }
      />

      <div className="px-6 py-6 pb-24 flex flex-col gap-6">
        {!cycleConfig ? (
          <EmptyState
            icon={<Activity size={48} />}
            title="Set up cycle tracking"
            subtitle="Configure your cycle schedule to unlock daily phase insights and the calendar view"
            ctaLabel="Get started"
            onCtaClick={() => setConfigOpen(true)}
          />
        ) : (
          <>
            <CycleDayWidget compact={false} />
            <CycleInsightCard />
            <CycleCalendarView />
          </>
        )}
      </div>

      <CycleConfigForm
        open={configOpen}
        onClose={() => setConfigOpen(false)}
        initialConfig={cycleConfig}
      />
    </PageTransition>
  )
}
```

## Implementation Notes

- `"use client"` directive is required — the page now holds local state (`configOpen`) and calls `useCycle()`. Previously it was a Server Component in T110; this change is intentional and correct.
- The `isLoading` guard (`if (isLoading) return null`) prevents a flash where Yara's EmptyState briefly appears while the auth check resolves for Yahya. Silent null is the right behavior — PageTransition handles the fade-in when content is ready.
- The `isOwner` split is the single, authoritative branch point. All cycle UI lives exclusively inside `if (isOwner)`. Do not scatter cycle-related code outside this branch.
- `CycleConfigForm` is only mounted in Yahya's branch — it is never part of Yara's render tree, not even as `open={false}`. This ensures no cycle-related markup ever reaches the DOM for Yara's session.
- The `rightAction` slot on `PageHeader` (T102) accepts a `ReactNode` — pass the gear button directly. If `PageHeader` does not yet have a `rightAction` prop, add it: `rightAction?: React.ReactNode` rendered at the end of the header row.
- `EmptyState` (T105) accepts an optional `ctaLabel` and `onCtaClick` for the setup CTA path — confirm these props exist in the component, or add them if not yet present. Do not add a new EmptyState variant; extend the existing one.
- Use `cn()` from `@/lib/utils` if any conditional class logic is needed on the content wrapper.
- All three cycle components (CycleDayWidget, CycleInsightCard, CycleCalendarView) have their own internal `isOwner` guards. Those guards are a safety net — do not rely on them as the primary gate. The page-level `isOwner` branch is the first and primary enforcement line.
- Do not add `Suspense` boundaries or loading skeletons to the cycle content area. The components handle their own null states when data is absent.

## Edge Cases

- **`isLoading` is true on initial render** → return `null` from the page. Do not show Yara's EmptyState as a default while resolving, because that would flash incorrect content for Yahya.
- **`isOwner` true but `cycleConfig` null** → show the setup CTA with the "Get started" button and keep the gear icon in the header. Tapping either one opens CycleConfigForm in first-time setup mode.
- **`CycleConfigForm` saves successfully** → `onClose` fires, `setConfigOpen(false)` runs, the Sheet closes. On the next render, `useCycle()` will return the newly saved `cycleConfig`, and the page will automatically transition from the setup CTA to the full dashboard — no manual state flip needed.
- **Yara somehow has `isOwner: true` due to a data bug** → the individual component guards (CycleDayWidget, CycleInsightCard, CycleCalendarView, CycleConfigForm) each independently return null. Defense in depth.
- **`PageHeader` rightAction prop missing** → check T102 implementation. If the prop does not exist, add `rightAction?: React.ReactNode` to the PageHeader interface and render it in the header's trailing slot. Do this as part of T507, not as a separate task.
- **`EmptyState` ctaLabel / onCtaClick props missing** → check T105 implementation. If these optional props don't exist, add them. The EmptyState renders a copper button below the subtitle when `ctaLabel` is provided.
- **Back button on PageHeader** — `backHref="/"` routes to the home screen, consistent with all other tab pages.
- **PWA safe area** — the `pb-24` on the content wrapper clears the BottomNav. Confirm this matches the BottomNav height defined in T101.

## Do NOT

- Do not render `CycleConfigForm` in Yara's branch — not even with `open={false}`. The component must not be in the DOM for Yara.
- Do not rely solely on the child component guards (`if (!isOwner) return null`) — the page-level `isOwner` branch is the primary and authoritative gate.
- Do not show a loading skeleton or spinner while `isLoading` is true — return null silently and let the content appear with the standard PageTransition fade.
- Do not use `router.push()` to open CycleConfigForm — it is a bottom sheet controlled by local `configOpen` state, not a route.
- Do not hardcode any color values — use Tailwind theme tokens tied to CSS variables.
- Do not add any copy that references "cycle", "period", "PMS", or any health-specific terms to Yara's EmptyState — it must match the neutral T110 placeholder exactly.
- Do not change the route (`/health`) — this is a modification of the existing page, not a new route.
- Do not add tabs, segments, or sub-navigation to this page — the layout is a single vertical stack.
- Do not use `left`/`right` CSS properties — use logical properties (`ps`, `pe`, `ms`, `me`) for RTL compatibility.
- Do not add a second `"use client"` boundary inside the file — the directive at the top of the file covers the whole module.
