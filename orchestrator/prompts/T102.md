# T102: PageHeader Component

Build the reusable page header component.

## Read First
- `CLAUDE.md`
- `docs/DESIGN_SYSTEM.md`

## What to Build

File: `src/components/shared/PageHeader.tsx`
Test: `src/__tests__/components/shared/PageHeader.test.tsx`

A page header bar that sits at the top of every page. It displays a title, an optional back button on the left, and an optional action slot on the right.

## Design Specs

- Height: 56px
- Background: transparent (inherits page background)
- Title: Playfair Display (`font-display`), 20px, `text-primary` (#2C2825), centered or left-aligned depending on whether back button is present
- Back button: `ChevronLeft` icon from Lucide, 24px, `text-secondary`, with `whileTap={{ scale: 0.9 }}` animation
- Right action slot: renders whatever ReactNode is passed (icon button, badge, etc.)
- Padding: `px-6 py-3`
- If `backHref` is provided, title shifts right. If no back button, title is left-aligned.

## Component API

```tsx
interface PageHeaderProps {
  title: string
  backHref?: string          // if provided, shows back arrow linking here
  rightAction?: React.ReactNode  // optional right-side element
  className?: string
}

export function PageHeader({ title, backHref, rightAction, className }: PageHeaderProps) { ... }
```

## Implementation Notes

- Use `"use client"` if using Framer Motion for the back button animation
- Back button uses `next/link` with the `backHref`
- Use flexbox: `flex items-center justify-between`
- Title uses `font-display` class for Playfair Display
- Right action area has min-width to balance the layout even when empty (so title stays visually centered when both back and right action exist)

## Test Requirements

- Renders title text
- Shows back button when `backHref` is provided
- Hides back button when `backHref` is not provided
- Renders right action when provided
- Back button links to correct href

## Do NOT

- Don't use `position: sticky` — the scroll container handles this
- Don't add a bottom border or shadow
- Don't import router for back navigation — use Link
