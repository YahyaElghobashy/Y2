---

## Why This Exists

Every page deeper than Home needs a name. The PageHeader is the chapter title of whatever you're looking at. It tells you where you are ("Us", "Health", "Settings"), lets you go back, and provides a slot for contextual actions (a gear icon, an add button, a filter). It's the orienting element вҖ” the thing your eyes hit first when a page loads вҖ” so it must feel authoritative but not heavy. Think: the title embossed on a journal's section divider.

---

## User Story

As a user on any page, I see a clean header at the top that tells me exactly where I am. If I'm deeper than the home screen, there's a back arrow that takes me to a sensible parent page. On some pages, there's an action button on the right (settings, add, filter). The header never competes with the content вҖ” it's subtle, elegant, and stays out of the way.

---

## Full UX Flow

1. **I navigate to /us** вҶ’ Header shows "Us" in Playfair Display serif. A back arrow on the left points to `/`. No right action for now.
2. **I tap the back arrow** вҶ’ subtle scale animation (0.9 on press), navigates to the backHref destination
3. **On /settings** вҶ’ Header shows "Settings" with back arrow to `/`. No right action.
4. **On Home (/)** вҶ’ No PageHeader is used. Home has its own greeting component (HomeGreeting in T108).
5. **Future: on a coupon detail page** вҶ’ Header shows "Coupon" with back arrow to `/us`, and a right-side "Edit" icon button.

The header does NOT scroll with content. It sits at the top of the page's scroll container. Content scrolls beneath it (but the header itself is within the content flow, not position:sticky вҖ” the pages handle their own scroll behavior).

- Background: transparent (inherits page bg, which is `bg-primary`)
- Title: Playfair Display (`font-display`), 20px, `font-bold`, `text-primary`
- Back icon: `ChevronLeft` from Lucide, 24px, `text-secondary`, with Framer Motion `whileTap={{ scale: 0.9 }}`
- Right action slot: any ReactNode вҖ” icon button, badge, text link, etc.
- No border. No shadow. No background color.
- Padding: `px-6 py-3`
- Layout: flexbox, `items-center`, `justify-between`

**Title positioning logic:**
- If back button exists AND right action exists вҶ’ title is centered between them
- If only back button вҶ’ title sits next to it (left-aligned with gap)
- If neither вҶ’ title is left-aligned with just padding
- Achieved by: back area and right area both have `min-w-[40px]` (even when empty) to balance the layout

## Implementation Notes

- `"use client"` if using Framer Motion for back button animation
- Back button: `next/link` wrapping the ChevronLeft icon
- Use flexbox with three sections: left (back or spacer), center (title), right (action or spacer)
- Both left and right sections have `min-w-[40px]` to maintain balance
- Title truncates with `truncate` class if too long (never wraps to two lines)
- The header is NOT position:sticky вҖ” it's part of the page content flow. Individual pages can make sections sticky if needed.

---

## Edge Cases

- **Very long title** (like "Notification History") вҶ’ truncate with ellipsis, single line only
- **Right action is complex** (badge + icon + text) вҶ’ right slot should handle flex layout internally
- **No back button, no right action** вҶ’ renders as simple left-aligned heading with padding
- **Title with Arabic text** вҶ’ should work fine with RTL, but for V1 we're English-first

---

## Do NOT

- Don't use `position: sticky` вҖ” not this component's job
- Don't add a bottom border or shadow вҖ” the header is transparent
- Don't use `router.back()` вҖ” always use explicit Link with href
- Don't render this on the Home page вҖ” Home has its own greeting
