# T801: Visual Audit Infrastructure

Create the screenshot infrastructure for automated visual auditing of UI components.

## Read First
- `PROJECT_CONTEXT.md` → `CLAUDE.md`

---

## Deliverables

### 1. `scripts/screenshot.mjs`

A Node.js script that uses Puppeteer to take headless screenshots:

- Import puppeteer via dynamic import (no npm install — uses npx cached binary)
- Accept args: `url` (default: http://localhost:3000), `output` (default: /tmp/y2-audit-home.png)
- Set viewport to 375x812 (iPhone, mobile-first per CLAUDE.md)
- Navigate with `waitUntil: 'networkidle0'`
- Take full-page screenshot
- Close browser
- Print output path to stdout

```js
#!/usr/bin/env node
const puppeteer = await import('puppeteer');
// ... implement
```

### 2. `scripts/visual-audit.sh`

A bash wrapper that:
1. Builds the app first (`npm run build` if not already built)
2. Starts `npx next start -p 3099` in background
3. Waits for port 3099 to accept connections (retry loop, max 15s)
4. Calls `node scripts/screenshot.mjs http://localhost:3099${ROUTE} ${OUTPUT}`
5. Kills the server process
6. Echoes the output path

Args: `$1` = route (default `/`), `$2` = output path (default `/tmp/y2-audit-home.png`)

Make it executable: `chmod +x scripts/visual-audit.sh`

### 3. `package.json` update

Add to scripts:
```json
"screenshot": "bash scripts/visual-audit.sh",
"screenshot:route": "bash scripts/visual-audit.sh"
```

---

## Constraints
- Do NOT add puppeteer to package.json dependencies or devDependencies
- Puppeteer auto-downloads via npx to ~/.cache/puppeteer/ (one-time ~170MB)
- Screenshots go to /tmp/ (ephemeral, auto-cleaned by OS)
- The build must already be complete before screenshots (script uses `next start`, not `next dev`)
- Must work headless on macOS without display server

---

## Test Requirements

File: `scripts/__tests__/screenshot.test.mjs` (or manual verification)

- `scripts/visual-audit.sh` is executable
- `scripts/screenshot.mjs` is valid Node.js (no syntax errors)
- Running `npm run screenshot` produces a PNG at the expected output path
- The PNG is non-zero size (actual image content)
