## Why This Exists

Love coupons are one of the most intimate features in Hayah. Creating one should feel like wrapping a present — deliberate, personal, a little exciting. This form is the creative moment: you're crafting something for your partner. It should never feel like admin. The emoji selector, the category pills, the surprise toggle — every element is designed to make the act of creating feel as warm as the coupon itself.

This form lives inside a bottom sheet. It slides up from the bottom of the screen when the user taps "Create Coupon" in the CouponWallet (T604). The form is the full creative surface — title, description, emoji, category, optional photo, optional expiry, and the special "surprise" mechanic that hides the coupon from the partner until revealed.

---

## User Story

As Yahya, I want to create a love coupon for Yara. I open the form, pick a heart emoji, type "Movie night — your choice", select the "fun" category, mark it as a surprise, and tap Create. It feels like I just wrapped a gift. Yara gets a push notification. When I reveal it later, she'll see it for the first time.

As Yara, I want to create a practical coupon — "I'll cook dinner, you pick the meal" — no emoji needed, no expiry, just a simple generous offer. I fill it in 20 seconds and it's there in Yahya's wallet.

## Full UX Flow

1. **CouponWallet taps "Create Coupon"** → bottom sheet opens, form mounts with slide-up animation
2. **User sees the form header** — "Create a Coupon / for [Partner Name]" — partner name from useAuth
3. **Emoji selector row** — scrollable horizontal row of suggested emojis, tap to select one; tapping again deselects; currently selected emoji is highlighted with a `bg-accent-soft` (#E8D5C0) ring
4. **Title field** — required, placeholder "e.g. Movie night", max 100 chars; character count shown subtly near the limit (>80 chars)
5. **Description textarea** — optional, placeholder "Any details? When? How to redeem?", max 500 chars; auto-expands up to 4 lines
6. **Category pills** — five pill buttons in two rows: [romantic] [practical] [fun] / [food] [general]; active pill has `bg-accent-primary` (#C4956A) text-white; inactive has `bg-accent-soft` text-secondary; default is "general"
7. **Surprise toggle** — toggle switch labeled "Mark as surprise"; when ON, info text fades in below: "Your partner won't see this coupon until you reveal it 🎁"; uses Framer Motion AnimatePresence for the info text entrance
8. **Photo upload** — tapping "Add photo" opens file picker (accept image/*); on selection, shows a 64×64px rounded thumbnail preview with an ×-to-remove overlay; no upload happens yet — the file is held in state
9. **Expiry date** — tapping "Set expiry date" reveals a date input (type="date") with min set to tomorrow; if a date is set, a small chip shows the formatted date with an × to clear it
10. **Create Coupon button** — copper filled button, full width, at the bottom; taps trigger validation first
11. **On submit:**
    - Button enters loading state (spinner, disabled)
    - `createCoupon()` from useCoupons hook called with form data
    - If photo file is present → upload to Supabase Storage at `coupons/{coupon_id}/photo`, then update the coupon row with `image_url`
    - On success → close sheet, fire `onSuccess()` callback, show toast "Coupon created! 🎁"
    - If `is_surprise` is false → partner receives push notification "You received a new coupon!"
    - On error → toast "Something went wrong, try again" and keep the form open with data intact
12. **Dismiss** — tapping outside the sheet or the × icon at top-right calls `onClose()` without saving

## Implementation Notes

- `"use client"` directive required — uses hooks, file input, state
- Use `useForm<CreateCouponFormValues>` from React Hook Form with `zodResolver(createCouponSchema)`
- Partner name: call `useAuth()` (T204) to get `partner.display_name`; fall back to "your partner" if unavailable
- `useCoupons()` (T602) exposes `createCoupon(data)` — call this on submit
- Image upload is separate from the RHF schema: use a `useState<File | null>` for the selected file; do NOT use `<input type="file">` inside RHF `register()` — manage it manually
- Upload flow: after `createCoupon()` resolves and returns the new `coupon_id`, call Supabase Storage `upload('coupons/{coupon_id}/photo', file)`, then call an `updateCoupon(coupon_id, { image_url })` helper from T602
- Emoji list constant at top of file: `const SUGGESTED_EMOJIS = ['😘', '❤️', '🎬', '🍕', '💆', '🎁', '🌹', '☕', '🥂', '🛁', '💌', '🎶']`
- Category list constant: `const CATEGORIES = ['romantic', 'practical', 'fun', 'food', 'general'] as const`
- Surprise info text: wrap in `<AnimatePresence>` — when `is_surprise === true`, render a `motion.p` with `initial={{ opacity: 0, y: -4 }}` `animate={{ opacity: 1, y: 0 }}` `exit={{ opacity: 0, y: -4 }}`
- Expiry date input: show/hide via a boolean `showExpiry` local state; clicking "Set expiry date" sets `showExpiry = true`; clearing the date chip sets `showExpiry = false` and calls `setValue('expires_at', undefined)`
- Photo preview: store selected file in `photoFile` state; generate object URL with `URL.createObjectURL(photoFile)` for the thumbnail; revoke on unmount or file change
- Bottom sheet: wrap entire form in shadcn/ui `Sheet` or `Drawer` component with `open` and `onOpenChange` wired to `onClose`; form content inside `SheetContent` with `side="bottom"` and a max height (e.g. `max-h-[90vh] overflow-y-auto`)
- `pb-[env(safe-area-inset-bottom)]` on the outermost inner div inside the sheet to avoid iPhone home bar overlap
- Reset form on close: call `reset()` from RHF and clear `photoFile` and `showExpiry` state in an effect when `open` changes to false

## Edge Cases

- **Title left empty** → Zod validation fires on submit; title field shows inline error "Title is required"; scroll to the field
- **Photo file too large** (>5MB) → show inline error below the photo row "Photo must be under 5MB"; do not proceed to upload
- **Photo upload fails after coupon is created** → coupon still exists without image; show toast "Coupon created, but photo upload failed. You can add it later." and call `onSuccess()`
- **createCoupon() network error** → toast error, form stays open, loading state clears, all data preserved
- **Partner name unavailable** (auth loading) → show "for your partner" as a skeleton shimmer or static fallback string
- **Expiry date in the past** → should not be possible (min="tomorrow") but if somehow submitted, Zod or backend will reject it; show toast "Expiry date must be in the future"
- **User taps outside sheet while form is partially filled** → `onClose()` is called; no confirmation dialog (keep it frictionless); data is lost — this is acceptable UX for this form
- **Rapid double-tap on Create** → button is disabled on first tap (loading state), prevents double submission
- **Very long title near 100 chars** → character counter appears at 80+ chars: small right-aligned span "87/100" in `text-secondary` below the input

## Do NOT

- Don't upload the photo before `createCoupon()` resolves — you need the `coupon_id` for the storage path
- Don't register the file input with React Hook Form's `register()` — manage the file separately in local state
- Don't use `router.push()` or navigate away on success — the sheet closes and the underlying CouponWallet refreshes
- Don't hardcode "#C4956A" or any hex values — use Tailwind CSS variable tokens only
- Don't add a drag-to-dismiss gesture manually — rely on the shadcn/ui Drawer/Sheet component's built-in behaviour
- Don't show the partner push notification from within this component — that is the responsibility of the `createCoupon()` hook/backend logic (T602)
- Don't block form submission if the Storage upload fails — coupon creation and photo upload are two distinct operations; a failed upload should never orphan the coupon
- Don't use `<img>` tags for the emoji — render them as plain text inside button elements
- Don't make the emoji row wrap to multiple lines — it must scroll horizontally on small screens
