## Why This Exists

The marketplace (T701) is built, but it's an island. No path leads there from within the app. A user would have to type the URL directly — that's not a product, that's a prototype. This task closes the loop: Yahya or Yara can now discover and reach the marketplace naturally from two places they already use every day — the CoYYns tab on the /us page, and the CoYYns widget on the home dashboard.

This is a pure wiring task. No new components. No new design patterns. Just a few well-placed links that feel like they were always there.

---

## User Story

As Yahya or Yara, when I'm looking at my CoYYns balance, I can see an obvious way to spend them. A single tap takes me directly to the marketplace. I don't have to hunt for it or remember the URL. The path feels natural: I see my balance, I see what I can buy, I go there.

---

## Full UX Flow

1. **User opens the /us page and taps the CoYYns tab**
   - They see the CoyynsWallet card showing their balance
   - Below the wallet (above CoyynsHistory), they see a row: `[ShoppingBag icon] Marketplace →`
   - Tapping the row navigates to `/us/marketplace`

2. **User opens the home dashboard and looks at the CoYYns widget (T308)**
   - Below the recent transactions in the widget, they see a link: `Shop with CoYYns →`
   - Tapping it navigates to `/us/marketplace`

3. **User is inside the marketplace**
   - The PageHeader back button (already built in T701) returns them to `/us`
   - No additional wiring needed here

4. **Tab bar stays at 4 tabs on /us**
   - Notes, Coupons, CoYYns, Send — no fifth tab added
   - Marketplace is a sub-page of the CoYYns experience, not a top-level destination

## Changes Required

### 1. CoYYns Tab on /us page

**File:** `src/app/us/page.tsx` (or wherever the CoYYns tab content is rendered — check T409)

Find the CoYYns tab panel. After the `<CoyynsWallet />` component and before `<CoyynsHistory />`, insert a marketplace link row:

```tsx
import Link from "next/link"
import { ShoppingBag } from "lucide-react"
import { cn } from "@/lib/utils"

// Inside the CoYYns tab panel, between CoyynsWallet and CoyynsHistory:
<Link
  href="/us/marketplace"
  className={cn(
    "flex items-center gap-2 py-3 px-1",
    "text-sm font-medium text-accent-primary",
    "transition-opacity hover:opacity-70 active:opacity-50"
  )}
>
  <ShoppingBag size={16} />
  <span>Marketplace →</span>
</Link>
```

- Font size: 14px (`text-sm`)
- Font weight: `font-medium`
- Color: `text-accent-primary` (copper #C4956A)
- Icon: `ShoppingBag` from Lucide React, 16px
- Layout: `flex items-center gap-2`
- No border, no card — just a clean inline link
- Vertical padding: `py-3` to make it easy to tap (44px minimum touch target)

### 2. CoyynsWallet Component — Optional Shop Link

**File:** `src/components/us/CoyynsWallet.tsx` (T303)

This is optional but high-value. If the wallet card has a header row (title + something), add a small "Shop" link in the top-right corner:

- Icon only, 16px, subdued opacity (0.6 default, 1.0 on hover)
- Positioned in the top-right of the card header via `justify-between` or `ml-auto`

## Implementation Notes

- Use `next/link` for all navigation — never `router.push()`
- Do not create new components — edit existing files only
- Use `cn()` from `@/lib/utils` for all class composition
- Use `text-accent-primary` token — never hardcode `#C4956A`
- Use `border-subtle` token — never hardcode the border color
- All touch targets must be at least 44px tall — use `py-3` padding on inline links
- RTL: use `gap-2` (logical) not `mr-2` / `ml-2` for icon-text spacing

---

## Edge Cases

- **CoyynsWallet has no header row** — skip the optional icon link in T303, rely entirely on the tab-level link below the wallet
- **CoyynsWidget has no transactions yet** — the "Shop with CoYYns" link should still appear; it is not conditional on transaction history existing
- **User is already on /us/marketplace** — back button returns to /us, not to root. No loop.
- **Link renders server-side** — `next/link` is valid in both server and client components. No `"use client"` directive needed just for these links unless the parent component already requires it.
- **Deep link to /us/marketplace from push notification** — back button must still go to /us. `backHref` prop handles this statically; do not make it dynamic.
