## Why This Exists

The coupon system has been built piece by piece across T601‚ÄìT608. The data layer, the hook, the cards, the wallet, the creation form, the detail sheet, the redeem dialogs, and the history feed are all complete and tested in isolation. They are waiting. This task is the moment they all connect ‚Äî the wiring that turns the Coupons tab from an EmptyState placeholder into the full, living coupon experience.

When Yara opens the /us page and taps "Coupons", she should see her wallet immediately. When she taps a coupon, the detail sheet opens. When she redeems it, Yahya gets a notification and sees a badge on the Coupons tab. When Yahya opens the app and taps "Coupons", the badge is there ‚Äî a small red dot that says "someone is waiting on you." Nothing new is being designed here. This task is purely about connection and flow.

---

## User Story

As Yara, when I open the /us page and tap the Coupons tab, I see my full coupon wallet. I can browse received coupons, tap one to see the detail, and redeem it ‚Äî all from one place. If I want to give Yahya a coupon, I tap "Create Coupon" and the form opens immediately. The whole experience is one continuous flow without any dead ends.

As Yahya, when I open the Coupons tab, I see a small red badge if Yara is waiting for my approval. I tap in, find the pending coupon, approve it, and the badge disappears. It takes ten seconds.

## Full UX Flow

**Coupons Tab ‚Äî Default View:**
1. User taps "Coupons" in the RelationshipTabs bar
2. `CouponsTabContent` renders ‚Äî mounts `CouponWallet`
3. CouponWallet shows "Received" tab by default with the user's received coupons
4. A "Create Coupon" button is visible at the bottom of CouponWallet

**Creating a Coupon:**
1. User taps "Create Coupon" in CouponWallet
2. CouponWallet fires `onCreateCoupon()` callback
3. `CouponsTabContent` sets `showCreateForm = true`
4. `CreateCouponForm` sheet slides up (open={showCreateForm})
5. On form success ‚Üí `onSuccess` callback calls `refreshCoupons()` and sets `showCreateForm = false`
6. On form close ‚Üí `onClose` sets `showCreateForm = false`

**Viewing a Coupon:**
1. User taps a coupon card in CouponWallet
2. CouponWallet fires `onCouponPress(coupon)` callback
3. `CouponsTabContent` sets `selectedCoupon = coupon`
4. `CouponDetail` sheet opens (open={selectedCoupon !== null})
5. User reads the full coupon detail ‚Äî emoji, title, description, photo, metadata
6. Tapping the backdrop or close button sets `selectedCoupon = null`

**Redeeming a Coupon (Recipient flow):**
1. Inside CouponDetail, recipient taps "Redeem This Coupon"
2. CouponDetail calls `redeemCoupon(coupon.id)` directly from `useCoupons()`
3. On success, CouponDetail calls `onStatusChange()` then `onClose()`
4. `onStatusChange` in `CouponsTabContent` ‚Üí calls `refreshCoupons()` + sets `selectedCoupon = null`
5. The creator (Yahya) receives a push notification and the Coupons tab badge appears

**Approving / Rejecting a Coupon (Creator flow):**
1. Inside CouponDetail, creator sees "Approve" and "Reject" buttons for a pending coupon
2. Tapping "Approve" ‚Üí `approveCoupon(coupon.id)` called directly within CouponDetail
3. Tapping "Reject" ‚Üí inline reject flow with textarea (handled inside CouponDetail)
4. On success, CouponDetail fires `onStatusChange()` then `onClose()`
5. `onStatusChange` in `CouponsTabContent` ‚Üí calls `refreshCoupons()` + sets `selectedCoupon = null`

**Pending Approval Badge:**
1. `useCoupons()` returns `pendingApprovals` ‚Äî an array of coupons where `status === 'pending_approval'` and `creator_id === me`
2. The "Coupons" tab label in the RelationshipTabs tab bar shows a small red dot badge if `pendingApprovals.length > 0`
3. Badge is a 8px circle (`bg-error`, rounded-full) positioned absolute at top-right of the tab label
4. Badge disappears immediately when the creator approves or rejects the last pending coupon

**Coupon History (Optional):**
- Below CouponWallet, render `CouponHistory` (T608) in compact mode
- Shows recent coupon events (created, redeemed, approved, etc.)
- CouponHistory manages its own data via the hook; no state lifted to the parent

## Component API / File Contracts

**File: `src/components/relationship/RelationshipTabs.tsx`**

The only file modified in this task. Add a `CouponsTabContent` private function (not exported) inside this file, replacing the Coupons tab EmptyState. Also update the tab label rendering to show the pending badge.

```tsx
// Private function ‚Äî not exported
function CouponsTabContent() {
  const { pendingApprovals, refreshCoupons } = useCoupons()
  const [showCreateForm, setShowCreateForm] = useState(false)
  const [selectedCoupon, setSelectedCoupon] = useState<Coupon | null>(null)

  return (
    <div className="flex flex-col gap-6">
      <CouponWallet
        onCreateCoupon={() => setShowCreateForm(true)}
        onCouponPress={(coupon) => setSelectedCoupon(coupon)}
      />
      <CouponHistory compact />
      <CreateCouponForm
        open={showCreateForm}
        onClose={() => setShowCreateForm(false)}
        onSuccess={() => {
          refreshCoupons()
          setShowCreateForm(false)
        }}
      />
      {selectedCoupon && (
        <CouponDetail
          coupon={selectedCoupon}
          open={selectedCoupon !== null}
          onClose={() => setSelectedCoupon(null)}
          onStatusChange={() => {
            refreshCoupons()
            setSelectedCoupon(null)
          }}
        />
      )}
    </div>
  )
}
```

The pending badge is added to the tab label for "Coupons" inside the existing tab bar render:

```tsx
// Inside the tab bar map / render ‚Äî update the Coupons tab label only
function CouponsTabLabel() {
  const { pendingApprovals } = useCoupons()
  return (
    <span className="relative">
      Coupons
      {pendingApprovals.length > 0 && (
        <span className="absolute -top-0.5 -end-2 h-2 w-2 rounded-full bg-error" />
      )}
    </span>
  )
}
```

Use `CouponsTabLabel` wherever the "Coupons" tab text was previously a plain string.

## Implementation Notes

- `"use client"` is already on `RelationshipTabs.tsx` ‚Äî no directive change needed
- `CouponsTabContent` and `CouponsTabLabel` are private helper functions in the same file ‚Äî do NOT export them, do NOT create separate files for them
- `useCoupons()` is called in both `CouponsTabContent` and `CouponsTabLabel` ‚Äî this is safe because the hook is backed by Supabase realtime and both will read from the same subscribed channel. React's reconciliation will not cause duplicate subscriptions as long as the hook follows standard `useEffect` cleanup patterns (T602 handles this)
- Do not lift `useCoupons()` to the `RelationshipTabs` level ‚Äî that would cause unnecessary re-renders on the entire tab component for every coupon state change
- `selectedCoupon` drives `CouponDetail`'s `open` prop via the `selectedCoupon !== null` check. The conditional `{selectedCoupon && ...}` prevents rendering the detail until a coupon is selected. When `onClose` sets `selectedCoupon = null`, both conditions go false together cleanly
- `onStatusChange` in `CouponDetail` is called *after* the status mutation succeeds and *before* `onClose`. The sequence: `onStatusChange()` ‚Üí `refreshCoupons()` ‚Üí `setSelectedCoupon(null)` ‚Üí CouponDetail receives `open={false}` ‚Üí exit animation plays. This order prevents a flash of stale content during the closing animation
- `CreateCouponForm`'s `onSuccess` and `onClose` both set `showCreateForm = false`. The distinction is: `onSuccess` also triggers a `refreshCoupons()` call. `onClose` (dismissed without submitting) does not refresh
- When the Coupons tab loses focus (user switches to another tab), do NOT reset `showCreateForm` or `selectedCoupon` ‚Äî unlike T409's CoYYns tab, these sheets layer on top of the content and can remain logically open while the tab is inactive. The sheet's own `open` prop controls visibility
- `CouponHistory` (T608) receives a `compact` prop ‚Äî check T608's actual prop name and align to it. If T608 uses `mode="compact"` instead, use that
- Import `Coupon` type from `@/lib/types/relationship.types` (added in T602)
- All imports go at the top of `RelationshipTabs.tsx` ‚Äî add only the new ones: `CouponWallet`, `CouponHistory`, `CreateCouponForm`, `CouponDetail`, and `useCoupons`

## Edge Cases

- **T608 not yet built** ‚Äî if `CouponHistory` is not ready when this task runs, render `null` where it would appear rather than blocking the rest of the wiring. Add a `// TODO: T608` comment. Do not stub it with an EmptyState
- **`pendingApprovals` changes while CouponDetail is open** ‚Äî if a realtime update resolves the pending state while the creator is looking at the detail, the CouponDetail will re-render with the updated coupon status. The action buttons will update correctly because CouponDetail reads directly from the `coupon` prop ‚Äî but the parent's `selectedCoupon` is a snapshot. If the coupon status changes, the displayed data could be stale. Mitigation: after `refreshCoupons()` resolves, if `selectedCoupon` is still open, update it by finding the refreshed coupon from the new `myCoupons` array by ID. This prevents the "Approve" button from reappearing on an already-approved coupon
- **`pendingApprovals.length > 0` on first mount** ‚Äî the badge must render correctly on initial page load, not just after a state change. Because `useCoupons()` fetches on mount, there may be a brief flash where the badge is absent. This is acceptable ‚Äî the initial fetch resolves quickly and the badge appears within the loading cycle
- **Both users on Coupons tab simultaneously** ‚Äî Supabase realtime in T602 ensures both clients get the status update. No special handling needed here; the hook re-fetches and state trickles down
- **No coupons at all** ‚Äî CouponWallet renders its own EmptyState (T604 handles this). `CouponsTabContent` does not need to know
- **Sheet open on top of sheet** ‚Äî `CreateCouponForm` and `CouponDetail` should not be open simultaneously. The UX flow does not allow it: you reach CreateCouponForm via the wallet (no coupon selected), and you reach CouponDetail by selecting a coupon (create form is closed). No guard code is needed, but do not set both `showCreateForm = true` and `selectedCoupon !== null` in the same handler
- **Tab switch while `refreshCoupons()` is in flight** ‚Äî the async call will still complete and update hook state, which is fine. No stale closure issue since `refreshCoupons` is a stable `useCallback` reference from T602
