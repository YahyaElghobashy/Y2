## Why This Exists

Love coupons are one of the most playful, personal features in Hayah. The CouponWallet is where that magic lives — it's the full collection, the drawer of sweet promises waiting to be redeemed. When Yara opens this view, she sees every coupon Yahya has created for her. When Yahya opens it, he sees what he owes and what he's earned. It needs to feel like flipping through a little book of joy, not like a task list.

This is not a generic list view. Every coupon carries emotional weight — a promise made between two people. The wallet should feel warm, organized, and easy to browse. Tabs keep "what I received" and "what I created" cleanly separated. Status filters let you quickly see what's still redeemable vs. what's already been used.

---

## User Story

As Yahya or Yara, when I open the Coupons section I see all my coupons neatly organized. I can switch between coupons I received and coupons I created. I can filter by status to see only active ones, or browse everything. Each coupon shows me exactly what I need at a glance. If I have no coupons yet, an encouraging empty state invites me to create the first one.

---

## Full UX Flow

1. **Screen opens** → "Received" tab is active by default, showing `receivedCoupons` from `useCoupons`
2. **Status filter chips** appear below tabs: All, Active, Redeemed, Pending — "All" selected by default
3. **Coupon list** renders filtered coupons using `StaggerList` wrapping individual `CouponCard` instances
4. **User taps "Created" tab** → list transitions to `myCoupons`, same filter chips apply
5. **User taps a status chip** → list filters immediately; if no results, `EmptyState` is shown
6. **User taps a coupon card** → `onCouponPress(coupon)` is called (parent handles detail/modal)
7. **User taps "Create Coupon"** → `onCreateCoupon()` is called (parent handles creation flow)
8. **Empty state** shown when: tab has no coupons, or active filter returns zero results

## Implementation Notes

- `"use client"` required — uses `useState` for active tab and active filter, imports Framer Motion
- Import `useCoupons` from `@/hooks/use-coupons` (T602) — provides `myCoupons`, `receivedCoupons`, `isLoading`
- Import `CouponCard` from `@/components/relationship/CouponCard` (T603)
- Import `StaggerList` from `@/components/shared/StaggerList` (T103) — wraps the coupon list
- Import `EmptyState` from `@/components/shared/EmptyState` (T105) — shown when filtered list is empty
- Tabs: use local `useState<'received' | 'created'>` — do NOT use URL state (this is a nested view)
- Tab indicator uses `motion.div` with `layoutId="coupon-wallet-tab-indicator"` for smooth slide
- Filter chips: `useState<CouponStatus | 'all'>` — filter applied client-side on the active tab's coupon array
- Filter logic: `coupons.filter(c => activeFilter === 'all' || c.status === activeFilter)`
- The chips row is `overflow-x-auto` with `scrollbar-hide` and `flex flex-row gap-2 pb-1`
- `StaggerList` receives the filtered array as children (map CouponCard inside)
- When `isLoading` is true, render a skeleton shimmer placeholder instead of the list (use LoadingSkeleton T104 if available, else a simple pulsing div)
- "Create Coupon" button is `fixed bottom-0` with `pb-[max(env(safe-area-inset-bottom),16px)]` — sits above the BottomNav, so add enough bottom offset (e.g. `bottom-20`)
- Use `cn()` from `@/lib/utils` for conditional classes throughout
- Coupon type `Coupon` imported from `@/types/relationship.types` or inferred from `useCoupons` return

## Edge Cases

- **No coupons at all (new user)** → EmptyState with gift icon, title "No coupons yet", subtitle "Create your first coupon for each other", CTA triggers `onCreateCoupon`
- **Has coupons but active filter returns zero** → EmptyState with filtered message, e.g. "No redeemed coupons" — CTA shows "See all" which resets filter to 'all'
- **`onCreateCoupon` not provided** → hide the "Create Coupon" button entirely, do not render it
- **`onCouponPress` not provided** → CouponCard press is a no-op; card should not show pointer cursor
- **Long coupon title overflows** → CouponCard handles this (T603); CouponWallet does not truncate
- **Tab switch while filter active** → preserve the filter selection across tab switches (don't reset to 'all')
- **Very many coupons (20+)** → list scrolls naturally within the page scroll; no virtualization needed at this scale

## Do NOT

- Don't use URL params or router state for tab/filter — this is a contained client component
- Don't hardcode color hex values — use Tailwind design tokens only
- Don't reset the filter when switching tabs — preserve filter state across tab switches
- Don't implement coupon creation or redemption logic here — this component is display + navigation only
- Don't add pull-to-refresh — that's a future enhancement
- Don't add sorting controls — coupons are ordered by the hook (newest first)
- Don't make the tabs horizontally scrollable — there are only two, they always fit
