## Why This Exists

The `/us` page is the heart of Hayah. It's the space where Yahya and Yara's relationship lives digitally — love notes, coupons, their CoYYns economy, and the ability to surprise each other with push notifications. This is the most emotionally loaded page in the app. For Phase 1, it's an empty shell with tabs — but those tabs need to signal what's coming. The empty states should feel like blank pages in a love letter book, not like a broken feature.

This is the page they'll visit most. It needs to feel special even when it's empty.

---

## User Story

As Yahya or Yara, when I tap the "Us" tab in the bottom nav, I arrive at our shared relationship space. Four tabs organize our interactions: Notes (love letters), Coupons (redeemable promises), CoYYns (our fake currency game), and Send (push notifications to each other). Each tab shows a beautiful empty state inviting me to create the first piece of content. Switching tabs feels smooth — content crossfades, the copper underline slides between tabs.

---

## Full UX Flow

1. **Tap "Us" in BottomNav** → page loads with PageTransition (fade + slide up)
2. **PageHeader** shows "Us" with back arrow to `/`
3. **Tab bar** sits below the header — four tabs evenly spaced:
   - **Notes** (MessageCircle icon) — love notes to each other
   - **Coupons** (Ticket icon) — redeemable love coupons
   - **CoYYns** (Coins icon) — fake currency dashboard
   - **Send** (Bell icon) — notification builder
4. **Active tab** has copper text + copper underline that slides between tabs (Framer Motion `layoutId`)
5. **Tab content area** shows an EmptyState for the selected tab. Content crossfades when switching tabs.
6. **Default tab** on page load: "Notes"

## Files to Build

### `src/components/relationship/RelationshipTabs.tsx`

The tab bar + tab content container. This is the main interactive component.

```tsx
"use client"

export function RelationshipTabs() { ... }
```

**Tab bar design:**
- Container: full width, `border-b border-border-subtle`, `px-6`
- Each tab: `flex-1`, center-aligned, `py-3`
- Tab content: Lucide icon (16px) + label text (13px, DM Sans, `font-medium`)
- Icon and label stacked vertically with tiny gap, OR side by side — your call, but side-by-side saves vertical space. Use side-by-side: icon 16px + label 13px, `gap-1.5`, centered.
- Inactive tab: `text-secondary` (#8C8279)
- Active tab: `text-accent-primary` (#C4956A)
- Underline indicator: 2px height, `bg-accent-primary`, width matches the tab, slides with `layoutId="us-tab-indicator"`, 250ms ease-out
- Tab bar should NOT scroll — all 4 tabs visible at all times

**Tab content:**
- Below tab bar, `px-6 py-6`
- Wrapped in `AnimatePresence mode="wait"` from Framer Motion
- Content uses `motion.div` with key={activeTab}, `initial={{ opacity: 0 }}`, `animate={{ opacity: 1 }}`, `exit={{ opacity: 0 }}`, duration 200ms
- Each tab shows an EmptyState component with tab-specific content:

| Tab | Icon | Title | Subtitle | CTA Label |
|-----|------|-------|----------|-----------|
| notes | `MessageCircle` (48px) | "No notes yet" | "Write your first love note" | "Write a note" |
| coupons | `Ticket` (48px) | "No coupons yet" | "Create one for your partner" | "Create coupon" |
| coyyns | `Coins` (48px) | "CoYYns wallet empty" | "Start earning together" | — (no CTA yet) |
| send | `Bell` (48px) | "Send a notification" | "Surprise your partner with a message" | — (no CTA yet) |

CTA buttons are present but non-functional for now (they'll link to creation flows in later phases). Use `onAction={() => {}}` as a no-op.

**State management:**
- `useState<"notes" | "coupons" | "coyyns" | "send">("notes")` — simple client state
- No URL params, no query strings — just local component state

## Edge Cases

- **Rapid tab switching** → AnimatePresence handles this, but ensure keys are unique per tab
- **Deep link to /us** → always opens on Notes tab (default state)
- **Browser back from /us** → goes to Home (`/`), handled by PageHeader's backHref
- **Tab names with different widths** → flex-1 ensures equal width regardless of text length

## Do NOT

- Don't use URL-based tab routing (no /us?tab=coupons) — client state is fine
- Don't build any actual feature content — only EmptyState placeholders
- Don't add a floating action button
- Don't make the tab bar scrollable — all 4 fit on screen
- Don't add pull-to-refresh
