## User Story

As Yahya or Yara, when I open the /us page and tap the CoYYns tab, I immediately see our shared balance and full transaction history. I can tap Add or Spend to open the relevant form. When I tap the Send tab, I see the notification composer ready to use, with my remaining daily sends shown inline, and my received notification history below. The app asks for push notification permission the first time I visit this page.

## Full UX Flow

**CoYYns Tab:**
1. User taps "CoYYns" tab on the /us page
2. CoyynsWallet renders at the top — shows shared balance and Add / Spend action buttons
3. Tapping "Add" → AddCoyynsForm sheet opens (controlled by `showAddForm` local state)
4. Tapping "Spend" → SpendCoyynsForm sheet opens (controlled by `showSpendForm` local state)
5. On form submit or dismiss → sheet closes, state resets to false
6. Below the wallet: CoyynsHistory renders in full list mode (not compact)
7. History updates reactively via the useCoyyns hook shared between wallet and history

**Send Tab:**
1. User taps "Send" tab on the /us page
2. NotificationBuilder renders — the full composer form
3. SendLimitIndicator is embedded inside NotificationBuilder (T406 handles its own placement)
4. When the user's send count is exhausted and they tap "Buy more" → BuyExtraSendModal opens
5. BuyExtraSendModal is controlled by `showBuyModal` local state in the Send tab
6. NotificationBuilder receives `onBuyMore` callback — it calls this when its internal limit UI triggers the purchase flow
7. Below NotificationBuilder: NotificationHistory in compact/received mode
8. On successful send → history refreshes reactively via useNotifications hook

**Notes Tab:** EmptyState placeholder remains. Future feature, not wired in this task.

**Coupons Tab:** EmptyState placeholder remains. Phase 6 (T609) will wire this.

**PushPermissionPrompt:**
1. Mounts in `us/page.tsx`, outside and above the tab component
2. Uses a `sessionStorage` flag (`push_prompt_shown`) to show only once per session
3. If `Notification.permission` is already `"granted"` or `"denied"` — does not render
4. If permission is `"default"` and the flag is not set — renders the prompt
5. On any user interaction (allow or dismiss) — sets the flag and unmounts

## Component API / File Contracts

**File 1: `src/components/relationship/RelationshipTabs.tsx`**

The existing tab switch in this file renders one of four content regions based on active tab. Replace the CoYYns and Send placeholder regions:

```tsx
// CoYYns tab region — replaces EmptyState
function CoyynsTabContent() {
  const [showAddForm, setShowAddForm] = useState(false)
  const [showSpendForm, setShowSpendForm] = useState(false)

  return (
    <div className="flex flex-col gap-4">
      <CoyynsWallet
        onAdd={() => setShowAddForm(true)}
        onSpend={() => setShowSpendForm(true)}
      />
      <CoyynsHistory />
      <AddCoyynsForm
        open={showAddForm}
        onOpenChange={setShowAddForm}
      />
      <SpendCoyynsForm
        open={showSpendForm}
        onOpenChange={setShowSpendForm}
      />
    </div>
  )
}

// Send tab region — replaces EmptyState
function SendTabContent() {
  const [showBuyModal, setShowBuyModal] = useState(false)

  return (
    <div className="flex flex-col gap-4">
      <NotificationBuilder
        onBuyMore={() => setShowBuyModal(true)}
      />
      <NotificationHistory mode="received" compact />
      <BuyExtraSendModal
        open={showBuyModal}
        onOpenChange={setShowBuyModal}
      />
    </div>
  )
}
```

## Implementation Notes

- Both `CoyynsTabContent` and `SendTabContent` are private functions inside `RelationshipTabs.tsx` — not exported, not registered as separate components
- `useCoyyns` (T302) is consumed inside CoyynsWallet and CoyynsHistory independently — no need to lift state; the hook handles shared cache via Zustand or SWR
- `useNotifications` (T402) is similarly consumed inside NotificationBuilder and NotificationHistory — no prop drilling needed
- `AddCoyynsForm` and `SpendCoyynsForm` are sheet-based (drawer/dialog) — they accept `open` and `onOpenChange` props to be controlled externally
- `BuyExtraSendModal` is a modal — same `open` / `onOpenChange` pattern
- `NotificationBuilder` receives `onBuyMore: () => void` — it calls this when its internal "buy more sends" button is pressed; the modal is owned by the parent tab content, not by NotificationBuilder itself
- `NotificationHistory` in Send tab uses `mode="received"` and `compact={true}` — check T408 props for exact names
- `"use client"` is already on RelationshipTabs (it manages tab state). No directive change needed.
- `us/page.tsx` remains a Server Component. PushPermissionPrompt is a Client Component — it renders as a client island within the server page tree.
- The sessionStorage key `push_prompt_shown` must be checked inside a `useEffect` (not during render) to avoid SSR mismatch

## Edge Cases

- **CoyynsWallet `onAdd` / `onSpend` props** — verify these callback names match the actual T303 component API. If the wallet uses different prop names (e.g. `onAddPress`), align to what T303 exports.
- **Sheet open on tab switch** — if a form sheet is open and the user switches tabs, the sheet should close. Reset `showAddForm` and `showSpendForm` to false when the active tab changes. Use a `useEffect` watching the active tab value.
- **BuyExtraSendModal already open** — if `showBuyModal` is true and a send completes (e.g. user bought more sends and sent immediately), close the modal. NotificationBuilder should call `onBuyMore` only once; no double-open risk.
- **Push permission already granted** — PushPermissionPrompt must not flash then disappear. Read `Notification.permission` synchronously in the effect before setting visibility state.
- **Notification API not available** — some environments (older Android browsers) don't have `window.Notification`. Wrap the permission check in `typeof Notification !== "undefined"`.
- **Empty CoyynsHistory** — CoyynsHistory renders its own empty state internally (T307). No extra handling needed in the tab.
- **Empty NotificationHistory** — same pattern; T408 handles its own empty state.
