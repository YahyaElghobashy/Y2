## Why This Exists

CoYYns are Yahya and Yara's shared fake currency — earned by streaks, habits, and challenges, spent on things that make their life together better. The Marketplace is where that currency becomes real. It's not a toy. When Yahya spends 25 CoYYns to send an extra notification, something actually happens. When Yara wins a challenge and watches her balance grow, she feels it.

This page is the economic heart of the Y2 system. It has two modes: the Shop (spend CoYYns on real items) and Challenges (earn CoYYns through bets and competitions). Both sides serve the same purpose — making the couple's shared game feel meaningful, not random. The shell established in this task is the entry point that houses both experiences.

---

## User Story

As Yahya or Yara, when I tap into the Marketplace from the /us page, I see my current CoYYn balance immediately in the top right corner. I can browse items I can buy with my coins in the Shop tab, or check active challenges in the Challenges tab. The page feels like a warm, intimate storefront — not a cold e-commerce site. I know exactly what I can afford and what I might earn.

## Full UX Flow

1. **Entry** → User arrives at `/us/marketplace` from the /us page (CoYYns section or dedicated button)
2. **Header renders** with back arrow (→ navigates to `/us`) and CoyynsBadge showing live balance on the right
3. **Default tab is "Shop"** — the list of purchasable items is immediately visible
4. **Shop tab content:**
   - V1 shows one real item: Extra Notification Sends (25 CoYYns each)
   - One or more "Coming Soon" placeholder cards below real items
   - Each ItemCard shows: icon, item name, short description, price in CoYYns
   - If user has insufficient CoYYns, the item's purchase action is visually disabled
5. **User taps "Challenges" tab:**
   - Tab indicator slides to Challenges
   - Content area transitions to the challenge list
   - Active challenges are shown (ChallengeCard components — built in T705)
   - A "Create Challenge" button sits at the bottom of the list
   - If no challenges exist, an EmptyState (T105) is displayed
6. **Tab switching** — animated with Framer Motion `layoutId` on the underline indicator, matching the BottomNav pattern
7. **Page transition** — wrapped in PageTransition (T103) for the standard fade + translateY entry
8. **Balance updates in real time** — CoyynsBadge reflects the Zustand store (T302); if a purchase happens elsewhere, this badge stays current

## Implementation Notes

- `"use client"` directive required — uses `useState` for tab, `useCoyyns` hook (T302)
- Route: `src/app/us/marketplace/page.tsx` — sub-route under `/us`, no new layout needed unless explicitly required
- Wrap the entire page content in `PageTransition` (T103) for entry animation
- Use `PageHeader` (T102): `title="Marketplace"`, `showBack={true}` (back navigates to `/us`), `rightAction={<CoyynsBadge />}` slot
- Tab switching: a local `useState<MarketplaceTab>` controls which content is shown; use Framer Motion `AnimatePresence` + `motion.div` with `key={activeTab}` to animate tab content in/out (fade only, 150ms)
- The tab underline uses `motion.div` with `layoutId="marketplace-tab-indicator"` positioned absolutely under the active tab label — same pattern as BottomNav
- ItemCard (T702) and ChallengeCard (T705) are separate components built in later tasks; this shell renders placeholder cards or imports them once available
- For V1, use inline placeholder cards inside the page until T702/T705 are built — stubbed as simple `div` blocks matching the visual spec
- `useCoyyns()` (T302) provides `balance` — pass it to CoyynsBadge or let CoyynsBadge read the store directly (follow T304 API)
- `ShopItem` type defined locally in this file for V1; migrate to `lib/types/marketplace.types.ts` when the backend is wired
- Do not fetch anything from Supabase in this task — all data is static; data layer comes in T703+

## Edge Cases

- **Zero CoYYn balance** → CoyynsBadge shows `🪙 0`; all purchasable items appear visually disabled (opacity-50, no pointer events) — do not hide them
- **Coming Soon items** → price displayed as `???`, card is non-interactive; no disabled styling needed, just opacity-60 on the entire card
- **Empty challenges list** → render EmptyState (T105) with icon `Trophy`, title `"No challenges yet"`, subtitle `"Create one to get started"`, and the "Create Challenge" CTA still appears below
- **Very long item name or description** → truncate at 2 lines with `line-clamp-2`; never let a card grow unbounded
- **Rapid tab switching** → `AnimatePresence mode="wait"` prevents overlap; `layoutId` on the indicator handles smooth repositioning
- **Back navigation** → `PageHeader` back button should use `next/navigation` `router.back()` as the handler so it respects the navigation stack; fall back to `/us` if no history
