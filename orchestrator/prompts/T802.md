### 3. Update `build_audit_prompt()` (line ~323-349)

Add STEP 6 after the existing STEP 5:

```
STEP 6 — Visual check (if UI component):
  npm run screenshot
  Then use the Read tool to view /tmp/y2-audit-home.png
  Check: Does the page render? Are colors warm (cream/copper tones)? Is spacing generous? Does typography look correct?
  If the screenshot fails or is blank, set visual_pass to false but don't fail the audit.
```

### 4. Update structured audit note (line ~933-945)

Add `visual:{'yes' if visual_ok else 'NO'}` to the header pipe-delimited format.

Extract `visual_ok` from `verdict.get("visual_pass", True)` alongside the other verdict fields.

### 5. Add cleanup after audit (after line ~961)

```python
# Clean up audit screenshots
import glob
for f in glob.glob("/tmp/y2-audit-*.png"):
    try:
        os.remove(f)
    except OSError:
        pass
```

### 6. Update CONFIG_FLOORS (line ~755)

Add or raise `"max_audit_minutes": 5` (raise floor from 3 to 5 to accommodate screenshots).

## Constraints
- Visual check is informational — a failed screenshot should NOT set approved=false
- The `visual_pass` field defaults to `true` if screenshots can't be generated
- Keep the audit prompt concise — one short step, not paragraphs
- Screenshots are deleted after every task audit (no accumulation)
- The `glob` import should go at the top of the file with other imports
