# T101: BottomNav Component

Build the bottom navigation bar for the Y2 PWA.

## Read First
- `CLAUDE.md` (project root)
- `docs/DESIGN_SYSTEM.md` (colors, radii, animation)
- `docs/CODING_STANDARDS.md` (component patterns, file naming)

## What to Build

File: `src/components/shared/BottomNav.tsx`
Test: `src/__tests__/components/shared/BottomNav.test.tsx`

A bottom navigation bar with 5 tabs. It stays fixed at the bottom of the screen, respects iOS safe area (home bar), and has a sliding copper underline indicator that animates between active tabs.

## Tabs

| Label | Icon (Lucide) | Route |
|-------|---------------|-------|
| Home | `Home` | `/` |
| Us | `Heart` | `/us` |
| Health | `Activity` | `/health` |
| Spirit | `Sun` | `/spirit` |
| Ops | `CheckSquare` | `/ops` |

## Design Specs

- Background: `bg-elevated` (#FFFFFF)
- Height: 64px + safe area inset bottom (use `pb-[env(safe-area-inset-bottom)]`)
- Top border: 1px `border-subtle` (rgba(44,40,37,0.08))
- Shadow above: `0 -2px 12px rgba(44,40,37,0.04)`
- Icon size: 20px
- Label font: DM Sans, 11px, font-medium
- Inactive: `text-secondary` (#8C8279)
- Active: `accent-primary` (#C4956A)
- Active indicator: 24px wide, 2px tall copper bar (`accent-primary`) positioned below the active icon. This bar slides horizontally using Framer Motion `layoutId` or `motion.div` with `layout` prop.
- Tap animation: `scale(0.95)` on press, `scale(1)` on release, 100ms

## Component API

```tsx
// No props needed — reads current route from Next.js usePathname()
export function BottomNav() { ... }
```

## Implementation Notes

- Use `"use client"` directive (needs hooks)
- Use `next/link` for navigation
- Use `usePathname()` from `next/navigation` to determine active tab
- Use `framer-motion` for the sliding indicator and press animation
- The indicator should use `layoutId="bottomnav-indicator"` so it animates between positions
- Each tab item is a `Link` wrapped in a `motion.div` with `whileTap={{ scale: 0.95 }}`
- Use the `cn()` utility from `@/lib/utils` for conditional class merging

## Test Requirements

- Renders 5 tab items
- Each tab links to the correct route
- Active tab has correct styling (check for accent color class)
- All icons render (check for SVG elements)

## Do NOT

- Don't use `router.push()` — use `Link` components
- Don't hardcode colors — use Tailwind theme tokens (`text-accent-primary`, `text-text-secondary`)
- Don't add a background blur or glass effect
- Don't make the nav scrollable — all 5 items always visible
