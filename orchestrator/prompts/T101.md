## Why This Exists

The BottomNav is the single most-touched element in the entire app. Every session, every feature, every moment starts with a tap on this bar. It's not a UI component — it's the heartbeat of navigation. When Yahya opens Hayah at 11pm to check if he should get Yara chocolates tomorrow, his thumb hits this bar. When Yara opens it to redeem a love coupon, she taps here. It needs to feel physical, responsive, and warm — like pressing a real button on a beautiful device.

This is a PWA installed on iPhone home screens. The bottom of the screen is where the thumb naturally rests. The nav must respect the iOS safe area (home bar gesture zone) so nothing gets clipped or covered.

---

## User Story

As Yahya or Yara, when I open the app, I see a bottom navigation bar that lets me move between the five core areas of our shared life. The navigation feels tactile — pressing a tab gives immediate physical feedback. A warm copper indicator slides smoothly between tabs so I always know where I am. I never feel lost. I never have to think about how to get somewhere.

---

## Full UX Flow

1. **App opens** → BottomNav is visible at the bottom of every authenticated page
2. **Five tabs are always visible**, evenly spaced:
   - **Home** (House icon) → `/`
   - **Us** (Heart icon) → `/us`
   - **Health** (Activity/pulse icon) → `/health`
   - **Spirit** (Sun icon) → `/spirit`
   - **Ops** (CheckSquare icon) → `/ops`
3. **The active tab** is highlighted:
   - Icon color changes from warm gray (`text-secondary` #8C8279) to warm copper (`accent-primary` #C4956A)
   - Label color changes identically
   - A small copper bar (2px tall, ~24px wide) sits directly below the active icon
4. **When I tap a different tab:**
   - My finger press immediately scales the tab item down (scale 0.95) — this is tactile feedback, it happens on press, not on release
   - On release, it scales back to 1.0
   - The copper indicator bar slides horizontally from the old tab to the new one — smooth Framer Motion animation using `layoutId`, 250ms, ease-out cubic
   - The old tab fades to gray, the new tab becomes copper
   - The page content transitions (that's handled by PageTransition in the page components, not by the nav)
5. **The nav is always present** — it doesn't hide on scroll, doesn't collapse, doesn't change
6. **iPhone safe area** — below the nav, there's padding that matches `env(safe-area-inset-bottom)` so the home bar gesture zone doesn't overlap with tap targets

## Implementation Notes

- `"use client"` directive required (uses hooks: `usePathname`, Framer Motion)
- Use `next/link` for all tab links — NOT `router.push()`
- Use `usePathname()` from `next/navigation` to determine which tab is active
- Match routes: `/` for Home, startsWith `/us` for Us, etc.
- The sliding indicator uses Framer Motion `layoutId="bottomnav-indicator"` — this makes it animate automatically between positions when the active tab changes
- Each tab item is a `Link` wrapped in `motion.div` with `whileTap={{ scale: 0.95 }}` and `transition={{ duration: 0.1 }}`
- Use `cn()` from `@/lib/utils` for conditional class merging
- Safe area: use `pb-[env(safe-area-inset-bottom)]` on the outermost container

---

## Edge Cases

- **App opens on /settings** (not a nav tab route) → Home tab should be active as default, or no tab highlighted. Prefer: no tab highlighted (honest about location).
- **User navigates via browser back/forward** → active tab must update reactively via `usePathname()`, not from click state
- **Rapid tab switching** → `layoutId` handles this gracefully, but ensure no animation queue buildup
- **Very small screen (iPhone SE)** → 5 tabs must fit without overflow or scrolling. Test at 320px width.
- **PWA installed vs browser** → safe area insets only apply when installed. Use `env()` with fallback: `pb-[max(env(safe-area-inset-bottom),8px)]`

## What This Connects To

- **AppShell (T107)** imports BottomNav and places it at the bottom of the layout
- **Every page** assumes BottomNav exists and adds `pb-24` (or similar) to avoid content hiding behind it
- **PageTransition (T103)** handles the content change animation — BottomNav doesn't animate page content

## Do NOT

- Don't use `router.push()` — Link components only
- Don't hardcode color hex values — use Tailwind theme tokens
- Don't add blur/glass/transparency effects to the nav background
- Don't make the nav scrollable — all 5 items are always visible
- Don't add badges or notification dots (that's a future feature)
- Don't add haptic feedback JS (CSS/animation handles the feel)
