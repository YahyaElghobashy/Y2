# T105: EmptyState — Beautiful Blankness

## Read First
- `PROJECT_CONTEXT.md` → `CLAUDE.md` → `docs/DESIGN_SYSTEM.md`

---

## Why This Exists

For weeks after launch, most pages in Hayah will be empty. No notes written yet. No coupons created. No CoYYns earned. The empty state IS the app experience for a while. If these empty pages feel hollow or broken, the app feels unfinished. If they feel warm and inviting — with a gentle nudge toward action — the app feels ready and waiting.

Every empty state should feel like the first page of a new journal: clean, inviting, full of possibility.

---

## User Story

As a user opening a section with no content yet, I see a friendly, warm placeholder that tells me what will go here and invites me to create the first piece of content. The placeholder has personality — not generic "Nothing to see here" energy, but specific, warm copy that makes me want to engage. There's an optional button that takes me directly to the creation flow.

---

## Full UX Flow

1. **I open /us → Notes tab** → No notes exist yet
2. I see: a message icon (muted, large), "No notes yet" as title, "Write your first love note" as subtitle
3. Below: a copper button "Write a note" — tapping takes me to the note creation flow
4. The whole thing is centered vertically in the available space, fading in softly

**Variations across the app:**
- Notes tab: MessageCircle icon / "No notes yet" / "Write your first love note"
- Coupons tab: Ticket icon / "No coupons yet" / "Create one for your partner"
- CoYYns tab: Coins icon / "CoYYns wallet empty" / "Start earning together"
- Health page: Activity icon / "Health tracking coming soon" / "Fitness, wellness, and more"
- Spirit page: Sun icon / "Spiritual practice coming soon" / "Prayer times, Quran tracker, and daily azkar"

Each page customizes the icon, title, subtitle, and optional CTA. The EmptyState component is the reusable shell.

## Component API

```tsx
// File: src/components/shared/EmptyState.tsx

interface EmptyStateProps {
  icon: React.ReactNode        // Lucide icon element, e.g. <Heart size={48} />
  title: string                // e.g. "No notes yet"
  subtitle?: string            // e.g. "Write your first love note"
  actionLabel?: string         // CTA button text, e.g. "Write a note"
  actionHref?: string          // if CTA is a link
  onAction?: () => void        // if CTA is a callback
  className?: string
}

export function EmptyState({ icon, title, subtitle, actionLabel, actionHref, onAction, className }: EmptyStateProps) { ... }
```

- Layout: `flex flex-col items-center justify-center`, `min-h-[300px]`
- Icon: passed as ReactNode, `text-muted` color (#B5ADA4), `mb-4`
- Title: DM Sans, 18px, `font-semibold`, `text-primary`, `mb-1`
- Subtitle: DM Sans, 14px, `text-secondary`, `mb-6`, `max-w-[240px]`, `text-center`
- CTA button (when `actionLabel` provided):
  - Background: `accent-primary` (#C4956A)
  - Text: white, 14px, `font-medium`
  - Padding: `px-6 py-2.5`
  - Border radius: `rounded-lg` (8px)
  - Hover: `scale(1.02)` + `shadow-soft`
  - If `actionHref` → render as `Link`
  - If `onAction` → render as `button` with `onClick`
  - If neither → no button renders
- Wrap everything in `<FadeIn>` from `@/components/animations` for soft entrance

---

## Edge Cases

- No subtitle provided → skip subtitle, button moves up
- No actionLabel → no button renders, just icon + title + subtitle
- Both actionHref and onAction provided → prefer actionHref (Link)
- Very long subtitle → wraps within max-w-[240px], stays centered
- Used in a very short container → min-h-[300px] ensures it takes up meaningful space

## Do NOT

- Don't hardcode any specific icon — always accept via prop
- Don't write generic copy like "Nothing here" — each usage provides its own copy
- Don't add illustrations or decorative images — icon only
- Don't make the CTA button outline/ghost — always filled copper
