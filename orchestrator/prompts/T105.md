# T105: EmptyState Component

Build the empty state placeholder component.

## Read First
- `CLAUDE.md`
- `docs/DESIGN_SYSTEM.md`

## What to Build

File: `src/components/shared/EmptyState.tsx`
Test: `src/__tests__/components/shared/EmptyState.test.tsx`

A centered, friendly empty state shown when a page or section has no content yet.

## Component API

```tsx
interface EmptyStateProps {
  icon: React.ReactNode        // Lucide icon component, e.g. <Heart />
  title: string                // e.g. "No notes yet"
  subtitle?: string            // e.g. "Write your first love note"
  actionLabel?: string         // e.g. "Write a note"
  actionHref?: string          // link destination for the CTA
  onAction?: () => void        // alternative: callback instead of link
  className?: string
}

export function EmptyState({ icon, title, subtitle, actionLabel, actionHref, onAction, className }: EmptyStateProps) { ... }
```

## Design Specs

- Layout: centered vertically and horizontally (`flex flex-col items-center justify-center`)
- Min height: 300px (so it fills space meaningfully)
- Icon: 48px, `text-text-muted` (#B5ADA4), `mb-4`
- Title: DM Sans, 18px, `font-semibold`, `text-primary`, `mb-1`
- Subtitle: DM Sans, 14px, `text-secondary`, `mb-6`, max-width 240px, text-center
- CTA button (if actionLabel provided): `accent-primary` background, white text, rounded-lg (8px), px-6 py-2.5, font-medium
- CTA hover: slight scale(1.02), shadow-soft
- CTA uses Link if `actionHref`, button with `onClick` if `onAction`
- Wrap the whole thing in a FadeIn animation (import from `@/components/animations`)

## Implementation Notes

- `"use client"` if using FadeIn wrapper
- Icon is passed as a ReactNode so the parent controls the icon choice
- If neither actionLabel, actionHref, nor onAction is provided, no button renders

## Test Requirements

- Renders icon, title
- Renders subtitle when provided
- Renders CTA button when actionLabel provided
- CTA links correctly when actionHref provided
- CTA fires onAction when provided

## Do NOT

- Don't use a specific fixed icon — accept any icon as prop
- Don't make the copy generic ("Nothing here") — each usage will customize
- Don't add illustrations or images — icon only
