## Why This Exists

The Health module will eventually house Yahya's most private feature: the period tracker he maintains for Yara (she never sees it — it helps him be a better partner). It will also hold fitness tracking, eye health reminders, and wellness data. For now, it's a warm empty shell. But the empty state copy should hint at what's coming without revealing the period tracker to Yara (since both users see this page — the tracker itself will be conditionally rendered based on user ID in Phase 5).

---

## User Story

As either Yahya or Yara, when I tap "Health" in the bottom nav, I see a clean page with a friendly empty state telling me health and wellness features are coming. The page feels intentional — not broken or forgotten, just not built yet. The empty state is warm and inviting, not clinical.

---

## Full UX Flow

1. Tap "Health" in BottomNav → page loads with PageTransition
2. PageHeader: "Health" with back arrow to `/`
3. Content: EmptyState with Activity icon, warm copy
4. Nothing interactive — just a beautiful placeholder

---

## File to Build

### `src/app/health/page.tsx`

```tsx
import { PageHeader } from "@/components/shared/PageHeader"
import { PageTransition } from "@/components/animations"
import { EmptyState } from "@/components/shared/EmptyState"
import { Activity } from "lucide-react"

export default function HealthPage() {
  return (
    <PageTransition>
      <PageHeader title="Health" backHref="/" />
      <div className="px-6 py-8">
        <EmptyState
          icon={<Activity size={48} />}
          title="Your wellness, tracked"
          subtitle="Fitness goals, health reminders, and wellness insights — all in one place"
        />
      </div>
    </PageTransition>
  )
}
```

Note the copy: "Your wellness, tracked" — not "Coming soon" (which sounds like a broken feature). The subtitle gives a taste of what's planned without being specific about the period tracker.
