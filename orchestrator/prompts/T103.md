Three reusable animation wrapper components.

## PageTransition

Wraps page content. Fades in + slides up 8px on mount.

```tsx
interface PageTransitionProps {
  children: React.ReactNode
  className?: string
}

export function PageTransition({ children, className }: PageTransitionProps) { ... }
```

Animation:
- Initial: `{ opacity: 0, y: 8 }`
- Animate: `{ opacity: 1, y: 0 }`
- Duration: 250ms
- Easing: `[0.25, 0.1, 0.25, 1]` (design system default ease-out)

Use `motion.div` from `framer-motion`.

## FadeIn

Simple fade-in wrapper with configurable delay.

```tsx
interface FadeInProps {
  children: React.ReactNode
  delay?: number  // seconds, default 0
  className?: string
}

export function FadeIn({ children, delay = 0, className }: FadeInProps) { ... }
```

Animation:
- Initial: `{ opacity: 0 }`
- Animate: `{ opacity: 1 }`
- Duration: 300ms
- Delay: configurable

## StaggerList

Wraps a list of children, staggering their entrance.

```tsx
interface StaggerListProps {
  children: React.ReactNode
  staggerDelay?: number  // seconds between each child, default 0.05
  className?: string
}

export function StaggerList({ children, staggerDelay = 0.05, className }: StaggerListProps) { ... }
```

Uses Framer Motion `staggerChildren` variant on the parent, each child wrapped in a `motion.div` with fade + slide up.

## Implementation Notes

- All components use `"use client"`
- Export all three from a barrel file: `src/components/animations/index.ts`
- These are pure wrappers — they don't dictate layout, just animation
- Use Framer Motion's `variants` pattern for StaggerList

## Test Requirements

- PageTransition renders children
- FadeIn renders children
- StaggerList renders children
- Components accept className prop

## Do NOT

- No exit animations (we don't use AnimatePresence for page-level transitions in Next.js App Router)
- No spring physics — use tween with ease-out only
- No bounce or overshoot
