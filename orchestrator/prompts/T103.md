# T103: PageTransition + FadeIn + StaggerList — Motion Language

## Read First

## Why This Exists

Animation in Hayah is not decoration. It's the feeling of the app. Every page reveal, every list building up, every element appearing — it communicates something: "this is intentional, this is crafted, this is alive." The Warm Mineral design language says: everything decelerates. Nothing bounces. Nothing overshoots. Like physics — like a heavy book page settling into place.

These three components are the animation vocabulary that every other component in the app speaks. They're wrappers — they don't dictate layout or content, only how things appear.

---

## User Story

As a user navigating between pages, content doesn't just pop in — it fades in softly and rises slightly, like a curtain being drawn. Lists build up item by item with a satisfying stagger. Individual elements can fade in with configurable timing. Nothing is jarring. Everything feels like it was placed there with care.

---

## Three Components to Build

### 1. PageTransition

Wraps entire page content. Every page uses this as its outermost wrapper.

**What the user experiences:** Navigate to a new page → content fades in (opacity 0→1) while sliding up 8px (translateY 8→0). Duration: 250ms. Easing: deceleration curve. The effect is subtle — you almost don't notice it consciously, but removing it would make pages feel jarring.

```tsx
// File: src/components/animations/PageTransition.tsx

interface PageTransitionProps {
  children: React.ReactNode
  className?: string
}

export function PageTransition({ children, className }: PageTransitionProps) { ... }
```

**Implementation:**
```tsx
<motion.div
  initial={{ opacity: 0, y: 8 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.25, ease: [0.25, 0.1, 0.25, 1] }}
  className={className}
>
  {children}
</motion.div>
```

### 2. FadeIn

Simple opacity fade wrapper. Used for individual elements that should appear after the page loads — like widget cards, detail sections, secondary content.

**What the user experiences:** An element appears by fading from invisible to visible. Can be delayed so things appear in sequence without the full stagger machinery.

```tsx
// File: src/components/animations/FadeIn.tsx

interface FadeInProps {
  children: React.ReactNode
  delay?: number    // seconds, default 0
  duration?: number // seconds, default 0.3
  className?: string
}

export function FadeIn({ children, delay = 0, duration = 0.3, className }: FadeInProps) { ... }
```

**Implementation:**
```tsx
<motion.div
  initial={{ opacity: 0 }}
  animate={{ opacity: 1 }}
  transition={{ duration, delay, ease: [0.25, 0.1, 0.25, 1] }}
  className={className}
>
  {children}
</motion.div>
```

### 3. StaggerList

Wraps a list of items. Children appear one by one with a slight delay between each — the classic "items building up" effect.

**What the user experiences:** A list of cards/items appears not all at once, but sequentially — first item fades and slides in, then the second 50ms later, then the third, etc. Gives the page a sense of being constructed in real-time.

```tsx
// File: src/components/animations/StaggerList.tsx

interface StaggerListProps {
  children: React.ReactNode
  staggerDelay?: number // seconds between each child, default 0.05
  className?: string
}

export function StaggerList({ children, staggerDelay = 0.05, className }: StaggerListProps) { ... }
```

**Implementation:** Uses Framer Motion variants pattern:
- Parent `motion.div` has variants with `staggerChildren: staggerDelay`
- Each child is wrapped in a `motion.div` with item variants: `{ hidden: { opacity: 0, y: 6 }, visible: { opacity: 1, y: 0 } }`
- Parent initial="hidden" animate="visible"
- Use `React.Children.map` to wrap each child in the motion item wrapper

## Animation Rules (from Design System)

- **Easing:** Always `[0.25, 0.1, 0.25, 1]` (ease-out deceleration) — things slow down as they arrive, like setting a book on a table
- **Duration:** 200-300ms for most transitions. Never over 500ms.
- **Direction:** Always upward (positive y to zero) or fade. Never sideways, never downward, never rotation.
- **NEVER:** bounce, spring with overshoot, wobble, parallax, or auto-loop

---

## Edge Cases

- **Fast navigation** → if user navigates before animation completes, the new page starts fresh. No exit animations (we don't use AnimatePresence at the page level in Next.js App Router because it conflicts with server components).
- **Reduced motion preference** → check `prefers-reduced-motion` media query. If true, skip animations (set duration to 0). Implement via: `const prefersReducedMotion = typeof window !== 'undefined' && window.matchMedia('(prefers-reduced-motion: reduce)').matches`
- **StaggerList with 0 children** → renders nothing, no error
- **StaggerList with 1 child** → renders normally, no stagger needed but still animates that one item

## What This Connects To

- **PageTransition** is used by: every page component (T108-T113 and beyond)
- **FadeIn** is used by: widget cards on home, detail views, secondary content
- **StaggerList** is used by: CoYYns history list, coupon wallet, notification history, settings rows
- These are the ONLY animation wrappers in the app. All entrance animations go through one of these three.

## Do NOT

- No exit animations (no AnimatePresence at page level)
- No spring physics — tween with ease-out only
- No bounce, no overshoot, no wobble
- No auto-playing or looping animations
- Don't import these as default exports — use named exports
